# Task Management

## Overview

### Feature Definition

BKN index build tasks are a core index management capability provided by the ADP BKN module. They support building object indexes for object classes in a **BKN**.  
With this feature, users can perform semantic knowledge queries, overcome the limitations of virtualization-based queries, improve query efficiency and recall, and quickly locate object instances that meet business conditions.

### Application Scenarios

This feature is suitable for **data analysts** who need to use semantic filtering to locate specific object instances in business analysis scenarios. For example:

- Finding enterprises whose names contain `chemical` or related similar concepts
- Filtering suppliers whose industry matches `new energy`

The object indexes generated by BKN index build tasks can quickly produce lists of objects that meet the conditions within a specified object class, providing support for later analysis such as confirming whether an object exists or checking its basic information.

## Key Terms

| Term | Meaning |
| --- | --- |
| Full build | A type of index build that creates indexes for all data of all target object classes |
| Incremental build | A type of index build that partially updates changed data with incremental identifiers |
| Task | The top-level unit of index building. A task can include a group of object classes or all object classes for unified management |
| Task status | A status that describes task progress, including `Pending`, `Running`, `Completed`, `Failed`, and `Canceled` |

## Prerequisites

Before using Task Management, make sure the following conditions are met:

- **Permission requirement**: You have access to the target BKN. The role permissions must be configured by an administrator in advance.
- **Data readiness**: The object classes to be indexed must already be bound to views. If a view is not bound, the system rejects the task creation request.
- **Task conflict check**: The object class must not already have a running task. If there is one, wait for it to finish or cancel it before creating a new task.

## Operation Guide

### Create an Index Build Task

#### Steps

1. Log in to ADP, open **BKN** from the workspace, and select the target BKN for which you want to create an index task.
2. In the left sidebar, click **Task Management**. After entering the page, click **Create Task** in the upper-right corner to open the dialog.
3. Configure the following parameters:
   - **Task name**: By default, the current BKN name is displayed. You can customize it. The maximum length is 40 characters, it cannot be empty, and duplicate names are allowed.
   - **Build mode**:
     - **Full build**: selected by default. Builds indexes for all data of all object classes.
     - **Incremental build**: updates only changed data that has incremental identifiers. If the data source is not configured with incremental identifiers, the system automatically performs a full build.
4. After configuration is complete, click **Confirm** to create the task.

#### Key Notes

- After a task is created successfully, the system automatically triggers it and runs it in parallel.
- After the entire task succeeds, the system replaces the bound indexes of the object classes in one step. If the task fails, the original indexes are retained and existing query capabilities are not affected.

### View the Task List

#### Steps

1. Open the **Task Management** list page and locate the target task.
2. In the actions column of that task, click **View** to check its details.

### Delete an Index Build Task

#### Steps

1. Open the **Task Management** page and locate the target task.
2. In the actions column, click **Delete**. If the task status is `Running`, the button is disabled.
3. The system displays a confirmation dialog stating that after the task is deleted, its related task details are also deleted and the operation cannot be restored. Click **Confirm** to complete deletion.

#### Key Notes

- Only tasks in the `Pending`, `Completed`, `Failed`, or `Canceled` states can be deleted.
- Deleting a task does not affect indexes that have already been built.

## Notes

- **Object class modification restriction**: When an object class has a running task, its configuration, such as bound views and field attributes, cannot be modified until the task finishes or is canceled.
- **Task operation support**: The current version does not support pausing running tasks. If you need to stop a task, click **Cancel**. Only tasks in the `Running` state can be canceled.
- **Task execution rules**: Tasks run in parallel. Execution efficiency depends on the system-wide coroutine count. Failure of one object class does not stop the entire task, but the task is marked as `Failed`.
- **Index replacement mechanism**: After a task is completed successfully, the system replaces the bound indexes of the object classes in one step. The replacement process does not interrupt ongoing business queries. If the task fails, the original indexes are retained and existing business use is not affected.

## FAQ

Q1: **The system says "Creation failed" when I create an index build task. How can I fix it?**  
A1: Check the following:

- The selected object class is not bound to a view: go to **BKN > BKN > Object Class Management**, locate the target object class, click **Bind View**, and configure the corresponding view.
- The selected object class already has a running task: go to **Task Management**, filter tasks in the `Running` state, check whether the target object class is included, then wait for the task to finish or click **Cancel** before creating the task again.

Q2: **If a task fails, what happens to the task and the existing indexes?**  
A2: Handle it as follows:

- Failure of one object class does not affect the rest of the task execution, but the task is marked as `Failed`.
- When a task fails, the original bound indexes of the object classes are not replaced. Check the cause of failure, such as abnormal view data or insufficient permissions, fix the issue, and then create the task again.

Q3: **Can I modify an existing index build task, such as changing the task name or adjusting the object classes?**  
A3: The current version does not support direct modification. Use the following approach instead:

- Delete the original task. Only tasks that are not running can be deleted.
- Recreate the task by following the steps in **Create an Index Build Task**.

Q4: **What impact does a successfully completed task have on later queries?**  
A4: The impact is as follows:

- After the task succeeds, the system replaces the original bound indexes with the new indexes. Later queries against those object classes automatically use the new indexes, improving efficiency and semantic recall.
- Index replacement runs in the background and does not interrupt ongoing queries, ensuring business continuity.
