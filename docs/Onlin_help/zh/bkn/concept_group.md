# 概念分组

**适用范围**：ADP 业务知识网络引擎模块，适用于业务分析师、数据工程师、AI 工程师等需管理 BKN 业务知识网络概念的用户。  
**相关模块**：业务知识网络引擎 > BKN


## 概述
### 功能定义
概念分组是 ADP 业务知识网络引擎中用于对 BKN 内**对象类、关系类、行动类**进行逻辑归类管理的核心功能。通过该功能，用户可根据实际业务场景，将分散的业务知识网络概念组织到不同分组中，支持单个概念归属多个分组（重叠管理），核心价值如下：
- **提升业务知识网络管理效率**：业务人员可按业务视角（如 “招聘”“客户服务”）快速定位相关业务知识网络，无需在全量业务知识网络中筛选；
- **加速业务知识网络模型上线**：实施人员可基于分组快速配置、验证业务逻辑，减少重复操作；
- **优化检索精准度**：缩小查询范围，帮助用户高效获取目标概念，尤其适用于复杂 BKN 场景。

### 应用场景
| 用户角色       | 场景描述                                                                 | 功能价值                                                                 |
|----------------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|
| 业务分析师     | 需求分析时，需查看某一业务视角（如 “招聘”）下的所有对象类、关系类         | 无需全量搜索，快速理解业务结构，提升需求分析效率                         |
| 数据工程师     | 为智能体配置数据源映射时，需批量处理同类对象类、关系类、行动类             | 支持批量导出 / 导入 / 修改业务知识网络配置，减少重复操作，降低配置失误率         |
| AI 工程师      | 构建 RAG（检索增强生成）系统时，需检索特定场景（如 “客户服务”）下的知识   | 限定分组检索范围，获取更精准的知识数据，提升智能体回答的准确性与相关性   |


## 前提条件
使用概念分组功能前，需满足以下条件：
1. 已登录 ADP，并进入 “业务知识网络引擎” 模块；
2. 已获得目标 **BKN** 的访问权限（由系统管理员配置）；
3. 执行**创建 / 修改 / 删除分组**等操作时，需具备对应的功能操作权限（具体权限说明可参考《ADP 权限管理指南》）。


## 操作指南
### 创建概念分组
1. 在 ADP 中，进入业务知识网络引擎后点击【BKN】，在 BKN 列表中，选择需创建分组的目标网络；
2. 进入目标 BKN 后，点击左侧导航栏【概念分组】，进入「概念分组」列表页；
3. 点击页面左上角【新建】按钮，弹出 “创建概念分组” 弹窗；
4. 在弹窗中按要求填写信息（带 “*” 为必填项）：
    ○ **名称***：输入分组名称，最多 40 个字符，不可为空，且需在同一 BKN 同一分支下唯一；
    ○ **ID**：可选自定义（仅支持小写英文字母、数字、下划线、连字符，且不可以下划线或连字符开头），未自定义则由系统自动生成，创建后不可修改；
    ○ **标签**：最多添加 5 个标签，每个标签不超过 40 个字符，不可包含 `# \ / : * ? " < > |` 特殊字符；
    ○ **备注**：可选，输入补充说明，最多 255 个字符；
5. 信息填写完成后，点击【确认】按钮，完成概念分组创建。

### 查看概念分组列表及详情
#### 查看分组详情
1. 进入「概念分组」列表页，找到目标分组；
2. 点击分组所在行任意位置，或操作列的【查看】按钮，弹出分组详情侧边栏；
3. 在侧边栏中可查看：
    ○ 基本信息：分组名称、ID、标签、备注、创建时间、更新时间；
    ○ 关联资源：对象类、关系类、行动类的数量及具体列表。

#### 说明
- 系统默认按 “更新时间” 降序（从新到旧）显示当前 BKN 下的所有概念分组；
- 列表中 “关联资源” 列会分别显示对象类、关系类、行动类的完整数量；
- 标签默认显示前 2 个，超过 2 个时显示 “+N”（N 为剩余标签数量），悬停 “+N” 可查看所有标签。

### 修改概念分组
1. 进入「概念分组」列表页，找到需修改的分组；
2. 点击该分组操作列的【编辑】按钮，弹出 “修改概念分组” 弹窗；
3. 在弹窗中修改信息（规则与创建时一致，**分组 ID 不可修改**）：
    ○ 名称：需保持在同一 BKN 同一分支下唯一，最多 40 个字符；
    ○ 标签：支持添加新标签（最多 5 个）或删除已有标签，需遵守标签字符规则；
    ○ 备注：可修改或删除已有内容，最多 255 个字符；
4. 修改完成后，点击【确认】按钮，系统弹出 “修改成功” 提示，完成分组修改。

### 删除概念分组
> ⚠️ 警告：删除分组后，该分组与所有业务知识网络概念的关联关系会自动解绑，分组信息及关联记录不可恢复；但不会删除业务知识网络概念本身，仅解除关联。

1. 进入「概念分组」列表页，找到需删除的分组；
2. 点击该分组操作列的【删除】按钮，弹出确认弹窗，提示：“您确定删除‘XX’概念分组吗？删除后使用该概念分组的功能将无法正常使用，请谨慎操作”；
3. 确认需删除时，点击【确定】按钮，系统弹出 “删除成功” 提示，完成分组删除；若需取消，点击【取消】按钮。

### 添加对象类到概念分组
> 说明：添加对象类后，其关联的关系类和行动类会自动关联至该分组，无需单独添加。

1. 进入「概念分组」列表页，点击目标分组，进入「分组编辑」页面；
2. 在页面左侧 “分组详情” 下，点击【对象类】选项卡，然后点击【添加】按钮，弹出 “选择对象类” 弹窗；
3. 在弹窗中选择需添加的对象类：
    ○ 左侧为全部对象类列表，支持通过 “搜索名称” 输入关键词筛选，或点击 “标签筛选” 下拉框单选标签筛选；
    ○ 勾选需添加的对象类（已添加至当前分组的对象类会显示为 “已选中”，不可重复勾选），勾选后会同步显示在弹窗右侧 “已选对象类” 列表中；
    ○ 若需移除已勾选的对象类，点击右侧列表中该对象类后的【×】按钮；
4. 确认已选对象类无误后，点击【确认】按钮，系统弹出 “添加成功” 提示，完成对象类添加。

### 移除分组中的对象类
> ⚠️ 警告：对象类移除后，其关联的行动类及所有悬挂关系类会一并永久移除，无法恢复，请谨慎操作；仅解除对象类与当前分组的关联，不会删除对象类本身。

1. 进入「概念分组」列表页，点击目标分组，进入「分组编辑」页面；
2. 在页面左侧 “分组详情” 下，点击【对象类】选项卡，查看当前分组下的对象类列表；
3. 勾选需移除的对象类（支持多选），点击列表上方的【移除】按钮，弹出确认弹窗，提示：“您确定移除选中的对象类吗？移除后其关联的行动类及悬挂关系类也会一并移除，且无法恢复”；
4. 确认需移除时，点击【确定】按钮，系统弹出 “移除成功” 提示，完成对象类移除；若需取消，点击【取消】按钮。

### 概念分组的导入与导出
#### 导出概念分组
1. 进入「概念分组」列表页，找到需导出的分组；
2. 点击该分组操作列的【导出】按钮，系统自动生成并下载 JSON 格式的导出文件（文件名称默认包含分组名称及导出时间）；
3. 导出文件包含：分组基本信息（名称、ID、标签、备注）及关联的业务知识网络概念（对象类、关系类、行动类）信息。

#### 导入概念分组
> 说明：导入仅支持单个概念分组（单个 JSON 文件），不支持批量导入；仅保存对象类与已存在分组的关联关系，若分组不存在则不保存该关系。

1. 进入目标 BKN 的「概念分组」列表页，点击页面右上角【导入】按钮，弹出 “导入概念分组” 弹窗；
2. 在弹窗中点击【选择文件】按钮，从本地选择需导入的 JSON 格式文件（系统会自动过滤非 JSON 格式文件）；
3. 文件上传后，系统会自动检测是否存在 ID 或名称重复的分组：
    ○ 若不存在冲突，直接进入导入流程，完成后弹出 “导入成功” 提示；
    ○ 若存在冲突，弹出 “冲突处理” 弹窗，选择冲突处理方式：
    ○ 【覆盖】：用导入文件中的分组信息替换系统中已存在的重复分组；
    ○ 【新建】：在系统中创建一个全新的分组，并将导入文件中的内容写入该分组中；
    ○ 【忽略】：保留系统中已存在的重复分组，跳过导入文件中的重复项；
    ○ 可勾选 “为之后所有的相同冲突执行此操作”，避免后续重复冲突时多次选择；
4. 选择冲突处理方式后，点击【确认】按钮，系统开始导入，导入完成后弹出 “导入成功” 提示（若有部分内容未导入，会显示未导入原因）。


## 注意事项
- 概念分组**不支持嵌套**（即一个分组不能包含另一个分组）；
- 关系类、行动类**不可直接添加 / 编辑 / 指定分组**，仅能通过关联的对象类间接关联至分组（添加对象类后自动关联，移除对象类后自动解绑）；
- 分组名称需在**同一 BKN 内唯一**，分组 ID 自定义时需符合字符规则，创建后不可修改；
- 标签最多添加 5 个，每个标签不可包含 `# \ / : * ? " < > |` 特殊字符，长度不超过 40 个字符；
- 删除对象类时，系统会自动解绑该对象类与**所有概念分组**的关联关系，关联记录不可恢复；
- 导入导出仅支持**单个概念分组**操作，导出文件为 JSON 格式，包含分组及关联概念信息。


## 常见问题
Q1：**为什么无法单独添加关系类或行动类到概念分组？**
A1：请按以下思路处理：
- 关系类、行动类依赖对象类（如 “员工 - 属于 - 部门” 关系中，“员工”“部门” 为对象类），单独添加无实际业务意义。
- 统一通过对象类关联，可避免关系类 / 行动类与分组的 “无效关联”，确保 BKN 的逻辑性。
- 若需添加关系类 / 行动类，只需添加其关联的对象类，系统会自动同步关联关系。

Q2：**导入概念分组时出现冲突提示，该如何选择处理方式？**
A2：可按以下方式选择冲突处理：
- 选择【覆盖】：导入文件为最新版本且需替换旧分组时使用，覆盖后原分组的基本信息与关联关系会被导入内容替换。
- 选择【新建】：在不影响现有分组的前提下保留导入内容时使用，系统将创建新分组并写入导入的配置与关联关系。
- 选择【忽略】：需保留系统中原有分组且仅导入非重复分组时使用，系统会跳过重复项，仅导入无冲突内容。
- 如存在多个重复冲突，可勾选 “为之后所有的相同冲突执行此操作”，避免重复选择。

Q3：**删除概念分组后，之前关联的业务知识网络对象类会被删除吗？**
A3：不会，具体说明如下：
- 删除分组仅解绑该分组与所有业务知识网络概念（对象类、关系类、行动类）的关联关系。
- 删除分组本身的信息（名称、ID、标签等）及关联记录。
- 业务知识网络对象类、关系类、行动类仍会保留在 BKN 中，可在全量业务知识网络列表中查询、使用或重新关联至其他分组。
