# 构建 BKN

## 概述
### 业务知识网络引擎定位
业务知识网络引擎以 “业务知识网络论” 为核心方法论，采用 “数据 - 逻辑 - 行动” 三位一体的业务知识网络模型，通过抽取对象类、关系类、行动类概念并建立关联，最终形成可支撑智能决策与自动化运营的 BKN。

### BKN 分层架构
BKN 按 “通用→行业→客户自定义” 纵向演进，各层级功能与定位如下：

| 层级                 | 核心作用                                                                 | 示例                                  |
| -------------------- | ------------------------------------------------------------------------ | ------------------------------------- |
| 通用 BKN             | 沉淀跨行业基础概念、关系与属性，提供标准化知识底座                       | 概念：“人员”“客户”；关系：“属于”“关联” |
| 行业 BKN             | 继承通用网络，补充行业专属概念与规则，降低垂直领域建模成本               | 金融领域：“风控指标”；零售领域：“供应链节点” |
| 客户自定义 BKN       | 支持企业按需定制个性化业务模型，实现场景精准映射                         | 特定产品的业务流程、客户分群策略      |


## 核心概念说明
BKN 的构建基于 “业务知识网络模型 + 资源”，核心要素定义如下：

| 要素       | 细分类型       | 定义                                                                 | 示例                                  |
|------------|----------------| -------------------------------------------------------------------- | ------------------------------------- |
| 业务知识网络模型   | 概念分组       | 对概念的逻辑归类，支持概念跨分组重叠                                 | “用户管理” 分组包含 “客户” 对象类、“关联订单” 关系类 |
|            | 概念 - 对象类  | 定义业务实体的本质属性（含数据属性与逻辑属性）                       | -                                    |
|            | 概念 - 关系类  | 描述实体间的结构性关联，不涉及业务动作                               | -                                    |
|            | 概念 - 行动类  | 定义业务操作的具体动作，含条件关联约束                               | -                                    |
|            | 映射 - 关联规则 | 建立概念与数据、逻辑、行动资源的绑定关系                             | -                                    |
| 资源       | 数据资源       | 来源于 VEGA 虚拟化的数据视图                                         | 客户消费记录视图、设备运行状态视图    |
|            | 逻辑资源       | 来源于 VEGA 数据模型、算子工厂的算子与函数                           | 客户活跃度计算模型、风险评分算子      |
|            | 行动资源       | 来源于算子工厂的工具与 MCP                                           | 短信推送工具、设备调度 MCP 接口      |


## 构建流程（实操步骤）
### 前提条件
1. 完成业务场景分析，明确需建模的核心实体与关联关系；
2. 已在 VEGA 虚拟化中创建所需数据视图（用于关联数据资源）。

### 创建 BKN
1. 登录系统，依次点击【业务知识网络引擎】>【BKN】；
2. 点击【+ 新建】，在弹窗中配置以下信息：
   1. **ID**：自定义标识（若不填写，系统自动生成）；
   2. **名称**：需体现业务场景（如 “零售客户管理 BKN”）；
   3. **标签**：用于分类和快速检索 BKN；
   4. **图标+颜色**：用于在系统中直观区分不同 BKN 的视觉标识；
   5. **描述**：简要说明网络用途（可选）；
3. 点击【保存】，完成创建。

### 定义概念（对象类 / 关系类 / 行动类）
#### 创建对象类
1. 进入【BKN】→「对象类」页面；
2. 点击【+ 新建】，按以下步骤配置：
   - **步骤一：数据视图**
     - 若在 DIP 中已有包含对象类数据的数据视图，选择【从数据视图导入属性】，系统自动映射数据视图列至属性（可在后续步骤丢弃）；
     - 若无现有数据源，选择【手动创建属性】继续。
   - **步骤二：基本信息**
     - **名称**：对象类的显示名称；
     - **ID**：对象类的唯一标识；
     - **图标 / 颜色**：配置默认图标和主题颜色，便于界面区分；
     - **概念分组**：将对象类归入指定分组，便于组织与筛选；
     - **描述**：简要说明对象类用途或含义。
   - **步骤三：属性定义**
     - 每个对象类至少需 1 个属性（作为主键唯一标识实例），可自定义添加其他属性；
     - 配置关键参数：
       - 主键：选择 1 个或多个属性组合（如 “客户 ID”），作为对象类实例的唯一标识；
       - 标题：指定 1 个属性作为对象类的默认显示名称；
       - 增量：仅支持整数（integer, unsigned integer）或时间类型（datetime, timestamp）。
   - **步骤四：属性映射**
     - 数据属性：与数据视图映射（基本类型属性），在画布中拖拽左侧逻辑视图字段至右侧逻辑属性；
     - 逻辑属性：与数据模型、算子、函数映射，需绑定资源 ID 并配置对象类数据属性与逻辑资源字段的关联（选择【逻辑属性】>目标属性【配置】）。
3. 点击【保存并退出】，完成对象类创建。

#### 创建关系类
1. 进入【BKN】→「关系类」页面；
2. 点击【+ 新建】，配置以下信息：
   - **名称**：体现关联含义（如 “客户 - 关联 - 订单”）；
   - **起点对象类 / 终点对象类**：选择需建立关联的两个对象类；
   - **关联方式**：
     - 直接关联：对象类属性与对象类属性直接连接（如 “客户 ID”→“订单表.客户 ID”）；
     - 间接关联：通过数据视图连接（适用于无法直接关联的场景，如 “设备”→“设备状态视图”→“故障记录”）；
3. 点击【保存并退出】，完成关系类创建。

#### 创建行动类
1. 进入【BKN】→「行动类」页面；
2. 点击【+ 新建】，配置以下信息：
   - **名称**：体现业务操作（如 “客户流失预警”）；
   - **条件关联**：设置行动触发规则（如 “客户近 30 天消费次数＜2 次”）；
   - **关联工具**：选择算子工厂中的目标工具（如 “短信推送工具”）；
3. 点击【保存并退出】，完成行动类创建。

### 配置索引
1. 进入「对象类」详情页，点击【操作】>【索引设置】；
2. 选择目标属性后，点击【设置】，在弹出的侧边栏中完成相关配置；
3. 点击【确定】，完成索引配置。

#### 索引配置规则
- 仅支持对 **string、text、vector** 类型字段进行索引配置，其他字段类型默认置灰，不可操作；
- **string / text 类型**支持配置以下索引能力：
  ○ 关键字索引：用于精确匹配，需配置字段长度；
  ○ 全文索引：用于文本检索，需选择分词器（standard / ik_max_word / english）；
  ○ 向量索引：选择小模型后，系统自动展示向量维度、批处理大小及最大 Tokens 数量；
- **vector 类型**：仅支持配置 **向量索引**。

### 导入 BKN
1. 在「BKN」列表页面，点击【导入】；
2. 上传历史配置文件（支持 JSON 格式）；
3. 系统自动解析并导入概念、映射关系，无需重新配置视图映射；
4. **注意**：导入后需按需创建索引构建任务，重新生成数据索引（持久化数据不保留）。

### 导出 BKN
1. 在「BKN」列表页面，选择目标 BKN，单击【更多】→【导出】；
2. 选择导出格式（JSON），点击【确认】下载。


## 附录
### 属性类型说明表
| 属性类型   | 说明                                                                 | 适用场景                     | 示例                                  |
|------------| -------------------------------------------------------------------- | ---------------------------- | ------------------------------------- |
| boolean    | 布尔值（真 / 假）                                                   | 状态标识、开关配置           | is_active: true                       |
| short      | 16 位有符号整数                                                     | 小范围数值、状态码           | age: 25                               |
| integer    | 32 位有符号整数                                                     | ID 标识、数量计数            | user_id: 12345                        |
| long       | 64 位有符号整数                                                     | 大数值 ID、时间戳            | timestamp: 1640995200000              |
| float      | 单精度浮点数                                                         | 精度要求不高的数值           | price: 99.99                          |
| double     | 双精度浮点数                                                         | 精确计算、地理位置           | latitude: 39.9042                     |
| decimal    | 高精度十进制数                                                       | 金融金额、利率计算           | amount: 12345.6789                    |
| varchar    | 可变长度字符串                                                       | 姓名、邮箱等通用文本         | name: "张三"                          |
| keyword    | 不分词字符串（精确匹配）                                             | 分类标签、状态码             | status: "completed"                   |
| text       | 分词文本（全文搜索）                                                 | 产品描述、文章内容           | description: "这是一段产品描述"        |
| date       | 日期（不含时间）                                                     | 生日、订单日期               | birthday: "1990-05-15"                |
| datetime   | 日期时间                                                             | 创建时间、更新时间           | created_at: "2024-01-20 14:30:25"     |
| timestamp  | 时间戳（毫秒 / 秒）                                                 | 系统日志、事件时间           | login_timestamp: 1642671025000        |
| vector     | 向量数据（相似度搜索）                                               | AI 特征向量、推荐系统         | embedding: [0.1, 0.2, 0.3]            |
| metric     | 指标数据（监控度量）                                                 | 系统监控、业务指标           | cpu_usage: 75.5                       |
| operator   | 操作符 / 函数引用                                                   | 逻辑计算、业务规则           | calculate_score（自定义评分函数）      |

### 分词器选择说明
| 分词器         | 适用语言 / 场景       | 特点                                           |
|----------------|-----------------------|------------------------------------------------|
| 标准分词器     | 多语言混合文本        | 按 Unicode 规则切分，自动转小写，通用性强       |
| IK 最大词分词器 | 中文文本              | 最大化保留词汇语义完整性，避免歧义             |
| 英文分词器     | 纯英文文本            | 支持词干提取（如 running→run），提升召回率     |
