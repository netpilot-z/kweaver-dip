# 任务管理

## 概述
### 功能定义
业务知识网络索引构建任务是 ADP 业务知识网络引擎提供的核心索引管理能力，支持为 **BKN** 中的对象类构建对象索引。  
通过该功能可实现基于语义的知识查询，弥补虚拟化查询的局限性，提升对象查询的效率与召回率，帮助用户快速定位符合业务条件的对象实例。

### 应用场景
适用于**数据分析师**在业务分析场景中，需通过语义筛选定位特定对象实例的场景。例如：
- 查找名称包含 “化工” 或相关相似概念的企业
- 筛选所属行业匹配 “新能源” 的供应商

通过业务知识网络索引构建任务生成的对象索引，可快速获取指定对象类中符合条件的对象列表，为后续分析（如确认对象存在性、查看基础信息）提供数据支撑。


## 关键术语定义
| 术语         | 含义                                                                 |
|--------------|----------------------------------------------------------------------|
| 全量构建     | 索引构建的一种方式，对所有对象类的全部数据进行完整索引构建           |
| 增量构建     | 索引构建的一种方式，针对具备增量标识的变更数据进行局部更新           |
| 任务         | 索引构建的顶层任务，可包含一组或全部对象类，用于统一管理多个任务     |
| 任务状态     | 描述任务执行进度的标识，包括：“待执行”“执行中”“已完成”“已失败”“已取消” |


## 前提条件
在使用任务管理功能前，需确保满足以下条件：
- **权限要求**：拥有目标 BKN 的访问权限（需管理员提前配置角色权限）。
- **数据准备**：需创建索引的对象类已绑定视图（无绑定视图时，系统会拒绝任务创建请求）。
- **任务冲突检查**：对象类无运行中的任务（若存在，需等待任务完成或取消后再创建新任务）。


## 操作指南
### 创建索引构建任务
#### 操作步骤
1. 登录 ADP，在工作台的「业务知识网络引擎」中点击【BKN】，选择需创建索引任务的目标 BKN。
2. 在左侧边栏点击【任务管理】，进入「任务管理」页面后，点击右上角【创建任务】按钮，弹出创建任务弹窗。
3. 在弹窗中配置以下参数：
   - **任务名称**：默认显示当前 BKN 名称，可自定义修改（最多 40 个字符，不能为空，支持重名）。
   - **构建方式**：
     - **全量构建（默认选择）**：对所有对象类的全部数据进行完整索引构建
     - **增量构建**：仅对具备增量标识的变更数据进行局部更新（若数据源未配置增量标识，则自动执行全量构建）。
4. 配置完成后，点击【确认】按钮，系统将完成创建任务。

#### 关键说明
- 任务创建成功后，系统会自动触发任务并行执行；
- 整个任务执行成功后，会一次性替换对象类的绑定索引；若任务失败，将保留原有索引，不影响现有查询功能。

### 查看任务列表
#### 操作步骤
1. 进入「任务管理」列表页，找到目标任务；
2. 在目标任务所在行的 “操作” 列，点击【查看】按钮，查看任务详情。

### 删除索引构建任务
#### 操作步骤
1. 进入「任务管理」页面，找到目标任务；
2. 在目标任务所在行的 “操作” 列，点击【删除】按钮（若任务处于 “执行中” 状态，【删除】按钮为灰色不可选）；
3. 系统弹出确认删除弹窗，提示 “删除任务后，其关联的任务详情将同步删除，且操作不可恢复”，点击【确认】按钮，完成任务删除。

#### 关键说明
- 仅支持删除 “待执行”“已完成”“已失败”“已取消” 状态的任务；
- 删除任务后，不影响已经创建好的索引。


## 注意事项
- **对象类修改限制**：当某对象类存在运行中的任务时，该对象类的配置（如绑定视图、字段属性）不允许修改，需等待任务完成或取消后再操作。
- **任务操作支持**：当前版本暂未实现 “暂停运行中任务” 的功能，若需终止任务，可点击【取消】按钮（仅 “执行中” 状态的任务支持取消）。
- **任务执行规则**：任务采用并行执行模式，执行效率受系统全局协程数量参数控制；单个对象类失败不会影响任务的运行，但任务会被标记为 “已失败” 状态。
- **索引替换机制**：任务成功完成后，系统会一次性替换对象类的绑定索引，替换过程不会中断业务查询，保障查询连续性；若任务失败，将保留原有索引，不影响现有业务。


## 常见问题
Q1：**创建索引构建任务时提示 “创建失败”，如何解决？**
A1：请按以下两项排查：
- 所选对象类未绑定视图：进入【业务知识网络引擎】>【BKN】>【对象类管理】，找到目标对象类，点击【绑定视图】，选择对应的视图完成配置。
- 所选对象类存在运行中的任务：进入「任务管理」页面，筛选 “执行中” 状态的任务，确认是否包含目标对象类，等待任务完成或点击【取消】后，再重新创建任务。

Q2：**任务失败后，对任务和现有索引有什么影响？**
A2：请按以下方式处理：
- 单个对象类失败不会影响其他任务，但任务会被标记为 “已失败”；任务失败后不会替换对象类的原有绑定索引，请排查失败原因（如视图数据异常、权限不足），修复后重新创建任务。

Q3：**能否修改已创建的索引构建任务（如修改任务名称、调整对象类）？**
A3：当前版本不支持直接修改，需按以下步骤操作：
- 删除原有任务（仅非运行状态任务支持删除）。
- 按照【创建索引构建任务】的步骤重新创建任务。

Q4：**任务成功完成后，对后续查询有什么影响？**
A4：请按以下说明理解：
- 任务成功后系统会用新索引替换原有绑定索引，后续基于该对象类的查询将自动使用新索引，效率与语义召回率提升。
- 索引替换在后台执行，不会中断正在进行的查询操作，业务连续性得到保障。
