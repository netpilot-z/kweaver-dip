# 如何导入导出 Decision Agent?

## 导入 Decision Agent
### 操作步骤
1. **进入导入入口**
    ○ 登录 ADP，点击【开发】>【Decision Agent】>【导入】按钮

2. **选择导入模式**
    ○ 点击【导入】按钮后，页面将弹出"导入模式"选择弹窗  
    ○ 选择导入模式：
    ○ **更新模式**：若系统中已存在相同标识的Decision Agent，则更新该Agent；否则创建新的Agent
    ○ **创建模式**：仅创建新的Decision Agent，若标识已存在则导入失败
    ○ 点击【确定】进入文件选择界面

3. **选择导入文件**
    ○ 在弹出的文件选择对话框中，选择本地的Decision Agent.json文件
    ○ 点击【打开】开始导入

4. **完成导入**
    ○ 系统将根据所选模式处理导入文件，导入成功后，可在【开发】>【Decision Agent】页面查看导入的Decision Agent


## 导出 Decision Agent
### 操作步骤
1. **进入导出入口**
    ○ 登录 ADP，点击【开发】>【Decision Agent】>【导出】按钮

2. **执行导出操作**
    ○ 点击【导入】按钮后，页面将进入批量选择模式（注：此处疑似表述误差，结合上下文应为“点击【导出】按钮后”）
    ○ 在列表中勾选您想要导出的Decision Agent（支持多选）
    ○ 选择完成后，点击【导出选中项】按钮

3. **完成导出**
    ○ 系统将自动生成一个包含所选Agent的.json格式导出文件，并下载到本地。


## 常见问题
Q1：**导入时提示“配置无效”怎么办？**
A1：请按以下步骤检查：
- 确认导入的 .json 文件是否被手动修改导致配置不符合系统要求。
- 建议使用系统导出的原始文件进行导入，避免因格式或字段变更引发错误。

Q2：**导入时提示“权限不足”怎么办？**
A2：请按以下步骤处理：
- 检查导入的 .json 文件是否包含“系统Decision Agent”配置。
- 若包含且当前账号无权限，请联系管理员获取相应权限后再导入。

Q3：**导入时遇到“Decision Agent 标识冲突”怎么办？**
A3：根据导入模式选择处理方式：
- 更新模式：若目标系统已存在相同标识且不属于当前用户，导入会失败，需更换目标或获取权限。
- 创建模式：若目标系统已存在相同标识，系统会自动生成新的标识以避免冲突。


## 注意事项
- 导入的Decision Agent不包含数据源信息，需要用户在导入后重新配置数据源连接。
- 系统内置的Decision Agent不支持导出，因为这些内置Agent在其他系统中已存在，且不允许修改。
- 导出文件为.json格式，包含了Decision Agent的所有配置信息，但不包含运行数据。
