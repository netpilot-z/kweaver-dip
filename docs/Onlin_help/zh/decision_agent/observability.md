# Decision Agent 可观测性（轨迹分析）

## 功能概述
- **定义**：智能体可观测性模块（又称"轨迹分析"）是为应用管理员提供的智能体运行监控与日志分析平台，用于追踪已发布智能体的执行情况、性能表现与问题定位。
- **功能目的**：通过多层级数据下钻，帮助管理员全面掌握智能体的运行状态、识别异常、优化性能，并为后续版本迭代提供数据依据。
- **目标用户**：应用管理员
- **功能位置**：属于DIP平台智能体治理与运维层，向上提供运行洞察，向下对接智能体执行引擎与日志系统。


## 典型使用场景
### 场景：应用管理员查看已发布智能体的运行表现，定位执行异常
- **触发条件**：业务用户反馈被智能体的处理结果异常，或管理员定期巡检智能体运行状况。
- **问题原因**：智能体在运行过程中可能出现性能下降、调用失败、逻辑错误等问题，需管理员介入分析。
- **用户目标**：管理员通过可观测性模块，从概览到详情逐层下钻，快速定位问题会话、分析执行链路、识别根本原因，并获取优化建议。


## 关键术语
| 术语 | 含义 |
| --- | --- |
| Decision Agent | 指具备决策能力的智能代理组件，可根据预设规则或算法执行特定业务任务，是可观测性功能的监控对象 |
| 运行指标 | 用于衡量 Decision Agent 运行状态的量化数据，包括 CPU 使用率、内存占用率、任务执行成功率、响应延迟等 |
| 运行日志 | 记录 Decision Agent 运行过程中关键操作、事件、报错信息的文本数据，包含时间戳、操作内容、状态结果等信息，用于问题定位 |
| 异常事件 | 指 Decision Agent 运行过程中出现的偏离正常状态的事件，如任务执行失败、资源占用超阈值、接口调用超时等，会触发相应提示 |
| 监控仪表盘 | Decision Agent 可观测性功能的核心展示界面，以图表（折线图、柱状图、仪表盘等）、列表形式直观呈现运行指标、异常事件等信息 |


## 功能结构与操作流程
### 1. 进入可观测性页面
- **操作入口**：登录 ADP，点击「开发」>「Decision Agent」>【新建】按钮/点击卡片 >「轨迹分析」Tab页
- **页面加载**：页面加载后，默认展示近一周内所有已发布Decision Agent的聚合运行数据与会话列表

### 2. 浏览首页监控数据
首页为您提供所有Agent运行状态的全局视角

#### 2.1 查看性能概览
- **数据展示**：页面顶部展示所选时间范围内所有智能体的核心性能概览数据
- **指标说明**：
    - 总请求次数：所有智能体被调用的累计次数 
    - 总会话数：产生的独立会话总数
    - 平均会话轮次：平均每次会话包含的交互轮数
    - 任务成功率：成功完成的会话比例
    - 平均执行耗时：会话从开始到结束的平均时长
    - 平均首Token响应耗时：从发起请求到收到首个响应的平均时间
    - 工具调用成功率：智能体调用外部工具或API的成功率

#### 2.2 对话日志列表
- **数据展示**：列表展示所选时间范围内产生的所有会话记录，每条记录代表一次完整的用户与智能体交互
- **信息概览**：您可以快速浏览每条会话的**会话ID、开始时间、结束时间、总运行时长以及状态**
- **核心操作**：点击任意会话记录中的【查看详情】按钮，即可进入该次会话的详情页，进行深度分析

#### 2.3 时间筛选器
- **功能说明**：您可以通过页面顶部的时间筛选器，灵活调整所要分析的数据时间范围
- **操作方式**：支持选择预设范围（如“今日”、“近7天”、“近30天”）或自定义起止日期。选择后，页面所有图表和列表将实时刷新

#### 2.4 获取优化建议（AI智能分析）
- **功能说明**：此功能利用AI对近期运行数据进行分析，为您提供潜在的优化方向和具体建议
- **使用流程**：
    1. 页面初始显示“暂无优化建议”及【AI智能分析】按钮
    2. 点击按钮后，系统基于近期的性能、错误日志等数据进行分析
    3. 分析完成后，面板将展示结构化的建议，可能涉及提示词优化、技能配置调整或资源分配等

### 3. 分析单次会话详情
当您需要对特定问题会话进行根因分析时，可下钻至本页面

#### 3.1 进入方式
- 在首页的「对话日志列表」中，点击目标会话记录旁的【查看详情】按钮

#### 3.2 查看会话详情
- **功能说明**：此页面提供选定单次会话的完整执行脉络和详细数据，用于问题定位与性能分析
- **核心信息**：
    1. **会话概览**：集中展示本次会话的ID、时间范围、总耗时和整体状态
    2. **执行步骤流水线**：按时间顺序清晰列出会话中的所有执行步骤（例如：意图识别、工具调用、生成回复等），并显示每个步骤的状态（成功/失败/进行中）与耗时
    3. **智能分析面板**：按时间顺序清晰列出会话中的所有执行步骤（例如：意图识别、工具调用、生成回复等），并显示每个步骤的状态（成功/失败/进行中）与耗时

### 4. 查看执行步骤详情
本页面提供了原子级步骤的透明化视图，是定位性能瓶颈或错误的关键

#### 4.1 进入方式
- 在「单次会话详情页」的「执行步骤流水线」中，点击任意步骤的【查看详情】按钮

#### 4.2 查看步骤详情
- **功能说明**：此页面展示一个具体执行步骤的详细运行情况，包括精确的性能数据和完整的输入输出信息，帮助您深入了解该步骤的执行过程
- **核心信息**：
    1. **步骤基本信息**：步骤名称、执行状态、精确的开始/结束时间戳、Token消耗量
    2. **执行时间线**：通过可视化时间轴，展示该步骤内部各子阶段（如网络请求、模型推理）的耗时分布，直观定位延迟环节
    3. **原始输入与输出**：完整展示该步骤接收的原始输入数据和该步骤执行后产生的输出结果
    4. **错误详情**：如果该步骤执行失败，页面会提供详细的错误代码、错误描述以及相关日志的查看链接，协助您进行问题排查


## 数据筛选与下钻逻辑
### 1. 按时间筛选数据
所有监控数据均基于您选择的时间范围进行统计和展示。您可以在首页顶部，方便地选择“今日”、“近7天”、“近30天”或自定义任意起止日期。切换时间范围后，页面上所有的性能图表和会话列表都将实时更新，确保您分析的是目标时间段内的准确情况。

### 2. 如何层层深入定位问题
平台设计了两级下钻路径，帮助您从发现异常现象快速定位到根本原因：

#### 第一层：从全局列表定位到问题会话
- 当您发现整体指标异常（如成功率下降）时，可以在首页的“对话日志列表”中，根据时间、状态等信息筛选，找到可能存在问题的具体会话记录。
- 点击该会话的【查看详情】按钮，即可进入该次会话的专属分析页面，查看其完整的执行过程。

#### 第二层：从会话内部定位到问题步骤
- 在会话详情页中，您可以逐一检查每个执行步骤的状态与耗时。
- 当发现某个步骤失败或耗时异常时，直接点击该步骤的【查看详情】按钮，即可查看这个步骤最详细的运行日志、性能分解和具体的输入输出数据，从而精准定位问题根源。


## 常见问题
### Q1：为什么进入可观测性页面后，所有图表都显示“无数据”？
- A1：请按以下步骤排查：
    1. 检查 Agent 状态：确认需要监控的 Decision Agent 已发布并正常运行。
    2. 调整时间范围：默认“近24小时”若无调用则无数据，尝试选择“近7天”等更长时间范围。
    3. 检查权限：确认账户具备 “运行指标查看” 权限，若缺失请联系管理员配置。

### Q2：为什么“任务执行成功率”显示为0%，但我确信有任务执行成功了？
- A2：请按以下步骤确认：
  1. 确认时间筛选范围覆盖成功任务的执行时间（如将“今日”调整为“近7天”）。
  2. 在“对话日志列表”中搜索目标时间段的任务，核实执行时间与筛选条件是否匹配。

### Q3：我可以将监控数据导出进行离线分析吗？
- A3：可以，按以下步骤操作：
  1. 在首页 “对话日志列表” 上方点击【导出】按钮。
  2. 系统会按当前筛选条件导出会话记录为 CSV 文件，便于离线分析、报告或存档。

### Q4：普通用户可以查看和处理异常事件吗？
- A4：请按以下说明理解：
  1. 查看：所有用户均可进入“异常事件列表”，对已处理的事件点击【查看记录】了解处理过程。
  2. 处理：仅应用管理员或具备指定运维角色的用户可受理、编辑、解决异常事件。

### Q5：“任务平均响应延迟”这个指标，是否包含了失败的任务？
- A5：请按以下说明理解：
  1. 该指标仅统计成功任务的平均响应延迟，旨在反映理想情况下的性能表现。
  2. 失败任务的耗时可在对应任务的“执行步骤详情页”查看，包括具体耗时与错误信息。
