# 测评数据集

## 概述
在评测场景中，评测集是一个精心设计的标准化测试数据集，用于系统化地效果评测。评测集通常包含输入样本和预期输出。输入样本作为评估对象的输入数据，预期输出提供了评估基准。在开始执行评测实验前，需要先准备好评测集。本文指导你在智能测评中创建和管理评测集。

**适用用户**：ADP 管理员、数据测评操作人员


## 评测集介绍
评测集是用于评测评估对象的一组数据。它通常包含输入数据和预期的输出结果，帮助开发者验证评估对象的效果。

对于评测集中通常包含以下列：
- 输入数据：这些是提供给评测对象的标准化测试样本，用于评测 AI Agent 在不同场景下的表现。
- 预期输出：这些是理想的结果，作为评估基准，帮助评估者或评估器对输出做出判断。


## 前提条件
已登录 ADP，且具备「智能测评」模块的访问权限（如无权限，请联系平台管理员申请）；


## 操作步骤
### 新建测评数据集
1. 登录 ADP，在工作台左侧导航栏选择智能评测中的【测评数据集】。
2. 在页面右上角单击【新建】。
3. 在弹窗中按要求填写信息（带 “*” 为必填项）：
    ○ **测评数据集名称 ***：输入具有辨识度的名称（如 “2025Q4 客户行为评测数据集”），长度不超过 50 个字符；
    ○ **csv文件解析规则 *：**配置文件解析方式，指定字段分隔符；
    ○ **描述：**简要说明数据集用途、包含的数据类型等（可选，长度不超过 200 个字符）；
    ○ **颜色*：**配置默认颜色，用于界面中的直观区分；
    ○ **测评数据集名称 ***：输入具有辨识度的名称（如 “2025Q4 客户行为评测数据集”），长度不超过 50 个字符；
    ○ **csv文件解析规则 *：**配置文件解析方式，指定字段分隔符；
    ○ **描述：**简要说明数据集用途、包含的数据类型等（可选，长度不超过 200 个字符）；
    ○ **颜色*：**配置默认颜色，用于界面中的直观区分；
4. 配置完成后，单击【创建】，进入「测评数据集」详情页。

#### 测评数据集版本管理说明
测评数据集支持**版本管理**，用于帮助数据驱动型团队在不同评测阶段持续迭代数据集，以提升评测质量和结果可靠性。
- 测评数据集**首次创建完成后，系统会自动生成第一个版本**；
- 系统默认生成该版本的**版本号**和**版本描述**；
- 用户可在页面右侧点击 **【编辑】**，修改版本名称和版本描述；
- 在此基础上，支持继续新建后续版本，用于数据集的持续迭代与评测。

#### 文件与版本管理
用于管理测评数据集中的测试文件及其所属版本。
- 进入目标测评数据集，切换至【文件与版本】页签，点击页面右侧【上传】上传测试文件或文件夹，上传内容将自动纳入当前版本管理；
- 对已上传的文件，可执行下载和删除操作，部分文件类型（如 CSV、JSON）支持在线预览，便于快速查看数据内容；
- 文件的新增或删除会影响当前版本的数据构成，相关变更需在版本层面统一维护。

#### 测评数据集设置
用于维护测评数据集的基础信息。
1. 进入目标测评数据集后，切换至 **【设置】** 页签；
2. 在设置页面中，用户可对测评数据集的**基本信息**进行维护，包括：
    ○ 修改测评数据集的**名称**，以更准确地反映其业务场景或评测用途；
    ○ 编辑测评数据集的**描述信息**，用于补充说明数据集内容、来源或使用说明；
3. 如测评数据集不再使用，用户可在设置页发起**删除操作**。系统将在删除前进行二次确认，删除后该测评数据集及其相关版本、文件将被永久移除，且**不可恢复**，请谨慎操作。
