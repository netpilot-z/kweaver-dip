# 效果测评任务

## 概述
### 功能介绍
在 Benchmark 中，任务用于将**评测规则**与**评估对象类型**进行组合并实际运行评测。  
- 评测规则：统一定义测试所使用的数据集与评测指标。  
- 评估对象：作为被测目标，参与同一规则下的评估过程。

### 应用场景
- 对比不同大模型在相同提示词下的响应准确率、生成速度等指标。  
- 验证 Agent 在特定业务场景下的任务完成率与稳定性。  
- 评估自定义应用或外部接入对象的接口调用成功率、数据处理效率。


## 前提条件
在创建效果测评任务前，请确保已完成以下准备工作：
1. 已登录 ADP，且具备「智能测评」模块的访问权限（如无权限，请联系平台管理员申请）；  
2. 已配置 **评估对象** 并通过校验（评估对象可为大模型、Agent，或通过 API 接入的外部算法）；  
3. 已完成 **测评规则** 的配置（评测规则包含可用的测评数据集与评测指标，且可被效果测评任务引用）。


## 操作步骤
### 新建测评任务
1. 登录 ADP，在工作台左侧导航栏选择「智能评测」中的【效果测评】；  
2. 在右侧页面中，切换至「效果测评任务」页签；  
3. 点击 **【新建】**，进入创建效果测评任务页面；  
4. 按要求填写下列信息（带 “*” 为必填项）：  
   - **任务名称***：用于标识当前效果测评任务的名称；  
   - **评测规则***：选择已配置并可用的评测规则（定义本次测评的数据集、指标维度及评分标准）；  
   - **评估对象类型***：选择参与测评的对象类型，可选项包括 **Agent**、**大模型**、**通过 API 接入的外部算法**；  
   - **选择评估对象**：指定参与本次测评的具体评估对象，支持同时选择多个对象进行对比评测；  
   - **上传 Adapter 文件**：适配测评数据集、评估对象与评测指标之间的输入输出格式，确保数据正确传递和解析；  
   - **描述**：补充测评任务的目的、使用场景等说明；  
   - **颜色**：为测评任务设置显示颜色，便于系统内视觉区分；  
5. 配置完成后，选择以下操作之一：  
   - **【保存】**：保存当前任务配置，不立即执行；保存后可后续编辑或手动运行；  
   - **【运行】**：保存配置并提交效果测评任务（立即执行或排队等待）。


## 任务管理操作
### 编辑任务
- 用途：修改已有任务的配置信息；  
- 操作：在目标任务的操作列中，选择【编辑】，进入任务配置页面调整设置并保存；  
- 注意：若任务已处于 “运行中” 状态，需先停止任务再进行编辑。

### 复制任务
- 用途：快速创建与现有任务配置相同的新任务；  
- 操作：在目标任务的操作列中，选择【复制】；系统基于原配置生成新任务，用户可修改后保存。

### 查看运行详情
- 用途：查看任务的执行结果与运行日志；  
- 操作：在目标任务的操作列中，选择【查看运行详情】，即可查看任务运行状态、评测结果及详细日志。

### 删除任务
- 用途：永久移除不再需要的任务；  
- 操作：在目标任务的操作列中，选择【删除】，按系统提示确认后删除；  
- 注意：仅 “已完成”“已停止” 状态的任务支持删除，“运行中” 任务需先停止再删除。
