# 如何建立数据标准
## 一、功能概述
数据标准管理用于建立统一的数据定义规范，避免不同系统、不同人员对同一数据产生理解偏差。

本模块核心包含四大内容：
- 数据元：定义数据的业务含义与属性标准
- 码表：定义字段取值范围及含义
- 编码规则：规范业务编码的生成方式和结构
- 标准文件：统一管理标准规范文档

### 建设价值
完成数据标准建设后，可实现：
- 数据口径统一
- 数据含义清晰
- 数据使用规范一致

支撑落地场景：
- 业务对象语义增强
- 数据质量规则配置
- 系统数据一致性

## 二、整体操作流程
标准搭建顺序：**定义数据元 → 配置码表 → 定义编码规则 → 关联标准文件**

补充说明：
- 数据元是标准体系的核心
- 码表和编码规则根据业务需要选择使用
- 标准文件用于统一管理规范说明

### 系统入口（操作路径）
「KWeaver DIP」>「全局业务知识网络」>「通用业务知识网络」>「数据标准」

## 三、具体操作指南
### 1、新建数据元
1. 进入路径：数据标准 > 数据元 > 点击【新建】
2. 填写信息：数据元中文名称、数据元英文名称、所属目录、所属部门、标准分类等
3. 点击【确定】，完成创建

#### 批量导入数据元
- 点击【导入】，下载系统标准导入模板
- 按模板要求完善数据元信息
- 上传填写完成的文件，完成批量导入

### 2、新建码表
1. 进入路径：数据标准 > 码表 > 点击【新建】
2. 填写信息：码表中文名称、码表英文名称、所属目录、所属部门、标准分类、码值及码值描述等
3. 点击【确定】，完成创建

#### 批量导入码表
- 点击【导入】，下载系统标准导入模板
- 按模板要求完善码表信息
- 上传填写完成的文件，完成批量导入

### 3、新建编码规则
1. 进入路径：数据标准 > 编码规则 > 点击【新建】
2. 填写信息：编码规则名称、所属目录、所属部门、标准分类、编码规则配置等
3. 点击【确定】，完成创建

### 4、添加标准文件
1. 进入路径：数据标准 > 标准文件 > 点击【添加文件】
2. 文件添加二选一：
   - 上传本地文件
   - 添加文件链接
3. 补充填写：所属目录、所属部门、标准分类、标准文件名称、标准编号、实施日期、说明等
4. 点击【确定】，完成文件添加
