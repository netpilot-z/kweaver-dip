# 如何查看与管理库表

库表管理用于对已接入系统的数据进行统一查看与基础语义治理，包括：
- 查看数据库表及字段结构
- 补充库表和字段信息
- 识别业务数据更新时间

通过库表管理，可以将「技术数据」转化为「可理解的业务数据」，为后续业务对象构建和数据治理提供基础。

## 一、前提条件
在使用库表管理功能前，请确保已满足以下条件：
- 已完成数据源接入
- 已对数据源发起数据扫描任务且扫描任务执行完成

> 注意
> 数据扫描完成后，系统才会生成对应的库表信息。如未完成扫描，将无法查看库表。

## 二、整体操作流程
库表管理主要操作流程：**查看库表 → 检查字段 → 完善/修正信息 → 保存 → 数据探查（可选）**

说明：
- 查看库表：确认数据是否已成功接入
- 完善/修正信息：完善字段及库表信息
- 数据探查：识别业务数据更新时间

### 入口路径
进入「**KWeaver DIP**」>「**全局业务知识网络**」>「**通用业务知识网络**」，选中「库表」即可进入库表管理页面。

## 三、详细操作说明
### 1、查看库表
进入库表页面后支持以下操作：
- 查看当前所有已扫描的库表列表
- 浏览每个库表的基础信息，支持按「组织结构、信息系统、数据源」三个维度筛选查看

点击任意库表名称或【详情】，可进入详情页查看：
- 库表信息
- 字段列表和字段详情
- 数据质量

### 2、编辑库表
点击【编辑/变更】按钮，进入库表编辑页面，可修改库表及字段信息。

#### 操作方式
- 双击库表/字段名称，可直接编辑
- 点击库表，在右侧弹窗修改库表详细信息
- 点击字段，在右侧弹窗修改字段详细信息

#### 异常字段处理
编辑中若出现字段异常（如名称重复），系统会自动标记：
- 点击【全部字段】筛选异常字段
- 对异常字段逐一修正

> 注意
> 所有异常字段处理完成后，才可更新库表信息。

#### 保存更新
完成字段与库表信息修改后，点击页面右上角【更新】，提交修改结果。

### 3、业务数据更新时间探查
业务数据更新时间用于标识数据的最新程度。

#### 操作方式
在库表列表选择【探查】>【发起探查】（业务数据更新时间探查）

#### 任务查看
- 探查任务发起后，系统弹出提示，可点击【探查任务】查看执行状态
- 关闭弹窗后，可在库表列表页上方点击【探查任务】查看全部任务

#### 结果查看
探查完成后，可在库表详情页查看最新的业务数据更新时间。
