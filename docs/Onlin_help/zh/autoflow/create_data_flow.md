# 数据流相关文档（Markdown格式）

## 概述
数据流指通过一系列**转换节点**，将数据从**源系统**自动化提取、转换，并最终**加载到目标系统**的数据处理过程。例如，在传统业务场景中，人工执行数据查询、分析、存储、转换等操作时，不仅耗时费力，还易出现人为错误。借助自动化数据流，可将上述数据处理过程设置为自动运行；同时，结合模型及智能体能力，还能进一步提升数据流的智能化水平，显著提高工作效率。


### 关键机制解析
- **数据驱动**：数据流的本质是由数据流动触发的自动化处理，而非简单的时间调度或人工指令。
- **ETL 逻辑**：遵循 Extraction（提取）、Transformation（转换）、Loading（加载）的标准范式。
- **写入为主**：区别于查询或展示，数据流的核心价值在于将处理后的数据**写入**到指定的存储或系统中。


### 数据流相关概念
1. **数据源**  
   数据源是数据管道的起点，提供原始数据，决定了数据的类型、质量和结构，是后续所有清洗、转换和分析的基础与原料。

2. **执行条件**  
   执行条件又称 “逻辑动作”，仅当满足预设条件时，系统才会执行后续操作，可理解为数据流中的 “分支”。
   - 分支运行顺序：按分支从左到右依次匹配，仅当前分支条件满足时，才执行该分支流程；一个分支内所有操作完成后，再执行分支外的操作。
   - 示例：“审核通过→文件上传” 可作为分支 1，“审核不通过→删除文件” 可作为分支 2。

3. **执行动作**  
   当触发动作发生且满足执行条件后，数据流自动执行的指定操作事件。
   - 示例：若满足 “审核通过” 条件，“文档在知识库上传发布” 即为数据流的执行动作。

4. **触发器**  
   触发器不是一个数据处理步骤，而是一个监控和决策机制。它持续监听一个或多个预设的条件或事件，一旦条件满足或事件发生，就立刻自动触发后续的数据流程。


## 创建数据流
创建数据流支持两种方式：**从空白新建**、**从模板新建**，具体操作步骤如下。


### 从空白创建数据流步骤
#### 基本流程
选择数据源 → 设置执行操作 → 设置触发器

#### 详细步骤
1. **进入新建页面**  
   进入「Autoflow」>「数据流」>「管道」后，点击【新建】，选择【从空白新建】，进入数据流新建页面。

2. **选择数据源**  
   配置需处理的数据，支持选择以下数据源类型：

| 数据源类型 | 说明 |
| --- | --- |
| 非结构化数据 | 无固定结构的数据（如文档、图片、音频等） |
| 结构化数据 | 具有固定格式与结构的数据（如用户组织信息） |
| 数据视图 | 基于一个或多个底层真实数据表（或其它视图），通过一条预定义的查询语句（如 SQL 的 SELECT 查询）动态生成结果集 |
| 算子数据源 | 通过 RESTful API 获取第三方数据（需提前配置算子） |

3. **设置执行操作**  
   执行操作用于处理或传递数据，同一流程中可设置 1 个或多个执行操作。
   - 支持功能：增加操作节点、设置分支及循环
     - 分支：默认添加 2 条分支，满足分支条件则执行对应分支；可设置 “且”“或” 逻辑关系；若不设置分支条件，将执行所有分支路径。
     - 循环：设置循环体内节点，循环输出结果可用于后续操作。

4. **详细设置（以 “执行操作 - 大模型” 为例）**  
   分为 “基础设置” 与 “高级设置” 两部分：
   - **基础设置**：配置执行动作所需的必要信息（如大模型选择、输入内容等）。
   - **高级设置**：自定义节点运行策略，支持配置超时时间、重试次数及重试间隔，以适配不同业务需求，提升任务执行稳定性。
     - **应用场景**：
       - 大数据量企业客户：当数据处理任务因数据量大、耗时长导致默认超时（1800 秒）频繁失败时，可延长超时时间并配置合理重试策略，确保任务完整执行。
       - 小数据量企业客户：数据处理任务耗时短，可缩短超时时间，快速获取结果，提升效率。
     - **配置规则**：
       - 取值范围：超时时间（1-86400 秒）、重试次数（1-10 次）、重试间隔（1-60 秒），超出范围将导致配置失败。
       - 流程兼容性：已存在的流程执行保持原有超时和重试逻辑；新创建或旧流程更新后，将遵循新配置。
       - 重试时间计算：固定间隔重试时，超时时间需≥最大重试次数 × 重试间隔（示例：最大重试 3 次、间隔 5 秒，总重试时间≤15 秒）。
       - 节点默认值：
         - 审核、图谱写入节点（异步节点）：前端不设置超时，后端默认超时 30 分钟。
         - Python 异步执行节点：支持异步 / 同步操作，默认超时 24 小时。
         - 其他同步节点：默认超时 30 分钟。

5. **设置触发器**  
   触发器是数据流的 “启动事件”，一个流程仅可配置 1 个触发器，且不可删除。
   - 特殊说明：当数据源为 “数据视图” 时，触发方式仅支持 “定时触发” 和 “手动触发”。


### 从模板创建数据流步骤
#### 前提条件
1. 已构建相关 BKN
2. 已构建将运用到的算子

#### 基本流程
选择 BKN → 设置执行操作 → 设置触发器

#### 场景示例：创建 “用户新增时自动更新 BKN” 数据流
该数据流的核心目标：当有新用户新增时，系统自动提取用户相关信息，更新至 BKN。

1. **进入模板选择页面**  
   点击进入「Autoflow」>「数据流」>「管道」后，点击【新建】，选择【从模板新建】，进入新建页面。

2. **配置 BKN**  
   点击【更新 BKN】，选择目标图谱（如“AS 默认图谱_1”）；系统将显示图谱内所有实体信息，根据需求选择目标实体；同时配置文档库（即文件储存的文件夹）。

3. **引入算子**  
   选择已配置好的算子，点击【确定】完成引入。

4. **完成数据流创建**  
   配置完成后，系统提示 “工作流创建成功”，完成从模板新建流程。

5. **验证数据流运行效果**  
   - 新增用户：进入「信息安全管理」>「用户」>「用户管理」页面，点击【新建用户】，完成用户创建。
   - 查看流程运行统计：返回数据流页面，点击【运行统计】，确认数据流是否正常触发。
   - 验证执行动作：选择执行动作，通过算子输出所有简历内容（若有相关配置）。

6. **数据流应用**  
   该数据流创建后将自动运行：当有新用户新增时，自动更新 BKN；用户通过超级助手提问时，系统可调用最新的用户信息生成回答。


## 重点 + 难点步骤详解
### 从空白创建：选择数据源（数据视图）
#### 关键配置说明
1. **数据获取方式**  
   数据视图支持两种数据获取逻辑，可根据业务场景灵活选择，具体差异如下表所示：

| 获取方式 | 定义 | 适用场景 |
| --- | --- | --- |
| 全量获取 | 基于用户设定的时间范围，一次性提取数据视图内的所有数据 | 需批量处理视图内历史数据或完整数据集的场景（例如：月度财务数据汇总、季度报表生成） |
| 增量获取 | 依托 “增量字段” 与自定义过滤规则，仅提取视图内新增或更新的数据。核心逻辑为：系统自动记录增量字段的最新值，后续仅获取数值大于该最新值的数据，并通过 SQL 条件进一步筛选 | 需实时或准实时同步新增数据的场景（例如：电商平台实时订单同步、用户行为日志增量采集） |

2. **增量字段设置**  
   - **字段类型限制**：仅支持将 “数据型字段”（如数字、日期时间类型）设置为增量字段，非数据型字段（如文本、布尔类型）不支持该配置。
   - **初始值含义**：初始值可根据业务需求自定义，若设置为 “0”，代表以 “初始状态” 或 “零值起点” 开始统计，适用于需从零计数的场景（例如：版本号从 0 开始迭代、任务状态从 0 标记为初始未执行）。

3. **特殊规则**  
   当数据视图的查询语言为**DSL（领域特定语言）** 时，数据获取方式默认固定为 “全量获取”，且界面不显示获取方式配置选项，无需手动调整。

4. **过滤规则配置**  
   支持通过自定义 SQL 语句的 WHERE 条件筛选数据，实现精准数据提取。  
   **示例**：筛选前一天 “区域编号为 275” 的分区数据，SQL 条件如下：  
   `"分区" = to_char( (CURRENT_DATE - INTERVAL '1 day') ,'yyyymmdd' ) and "区域" = '275'`

5. **批大小设置**  
   - **定义**：单次数据处理的最大数量阈值，系统会按照该阈值自动分批次循环处理数据，直至覆盖所有待处理数据。
   - **取值范围**：1000-10000（若输入值超出范围，系统会提示配置错误，需重新调整）。

6. **数据预览**  
   数据源配置完成后，点击 “数据预览” 按钮可实时查看数据格式、字段内容及数据量，帮助验证配置是否符合预期，避免因数据源错误导致后续流程失败。


### 从空白创建：选择数据源（算子数据源）
#### 关键配置说明
1. **算子前置要求**  
   - 需提前在系统 “行动” 模块完成目标算子的配置，配置成功的算子会自动同步至数据源选择列表，未配置的算子无法选择。
   - 算子必须添加 “数据源标识字段”，仅带有该标识的算子可在数据源节点中使用，无标识算子会被过滤隐藏。
   - 数据源算子仅限 “同步算子”，异步算子不支持作为数据源（因异步算子无法实时返回数据，会导致数据流阻塞）。

2. **变量使用限制**  
   数据源节点仅支持引用**全局变量**（例如：用户身份令牌 token、系统全局配置参数等），不支持引用流程内其他节点的局部变量，避免变量作用域冲突。


### 从空白创建：选择执行操作
#### 写入 BKN
- **功能描述**：支持将 “内置实体”（文件、用户、组织）或 “自定义实体” 的信息，写入指定的 BKN，实现实体数据与知识网络的同步更新。
- **配置步骤**：
  1. 在节点配置界面选择目标 BKN（需提前创建并授权访问权限）；
  2. 从网络内的实体列表中，选择需写入数据的实体类（例如：“用户” 实体、“订单” 自定义实体）；
  3. 配置实体属性值：可选择引用前序节点的输出变量（动态获取数据），或直接手动输入固定值（静态数据），属性值需与实体类的字段类型匹配（例如：“用户年龄” 字段需输入数字类型值）。


#### Python 代码执行
- **功能描述**：通过编写 Python 脚本，实现复杂逻辑的自动化处理（例如：数据清洗、格式转换、自定义算法计算等），支持灵活扩展数据流能力。
- **配置步骤**：
  1. **输入变量配置**：
     - 选择变量类型（支持 string 字符串、int 整数、array 数组、object 对象）；
     - 变量值来源：可引用前序节点的输出变量（例如：引用 “数据提取” 节点的 result 变量），或手动输入自定义值（例如：输入固定字符串 "test_data"）。
  2. **输出变量配置**：
     - 设置输出变量的类型（同输入变量类型），需与脚本的返回结果类型一致（例如：脚本返回数组，则输出变量类型需设为 array）。
  3. **代码编写**：
     - 在代码编辑区编写 Python 逻辑，输出结果必须使用 return 语句返回（不可使用 print 语句，print 仅用于日志打印，无法作为流程输出）；
     - **示例代码**：实现将输入的字符串转换为大写，代码如下：
     ```Plain Text
     def process(input_str):
         return input_str.upper()
     result = process({{input_var}})  # {{input_var}}为引用的输入变量
     return result
     ```
- **依赖库说明**：系统已内置常用 Python 库（例如：pandas、numpy、json 等），可直接导入使用；若需使用外部第三方库，需参考《在数据流中使用 Python 节点》文档，提前完成库的安装与配置。


#### JSON 处理
支持三种 JSON 操作类型，满足数据提取、编辑、格式化的核心需求，具体功能如下表所示：

| 操作类型 | 核心作用 | 工作流程 |
| --- | --- | --- |
| 根据 Key 获取值 | 从复杂 JSON 对象中精准提取指定 Key 对应的 Value，解决多层级 JSON 数据的取值问题 | 1. 输入目标 JSON 数据（可引用变量或手动输入）；2. 输入需提取的 Key（支持多层级 Key，例如："user.info.name"，表示获取 user 对象下 info 子对象的 name 值）；3. 系统自动在 JSON 中匹配 Key，输出对应的 Value（若 Key 不存在，输出 null）。 |
| 根据 Key 设置值 | 修改现有 JSON 的指定 Key 值，或为 JSON 新增 Key-Value 对，生成全新的 JSON 对象 | 1. 输入原始 JSON 数据（基础数据来源）；2. 输入需设置 / 修改的 Key（支持多层级 Key）及对应的 Value（Value 类型需与 JSON 结构兼容）；3. 系统自动更新 JSON 数据，输出修改后的新 JSON（原始 JSON 保持不变，避免影响其他流程）。 |
| 模板转换 | 将 JSON 数据填充至预定义的文本模板中，生成结构化文本（例如：报告、通知文案等） | 1. 输入 JSON 数据源（提供填充数据）；2. 编写文本模板，模板中需用占位符 {{Key}} 标记需填充的位置（例如："尊敬的{{user.name}}，您的订单{{order.id}}已发货"）；3. 系统自动将 JSON 中对应 Key 的 Value 注入模板占位符，输出渲染后的完整文本。 |


#### AI 能力调用
支持调用大模型、小模型（Embedding/Rerank）及智能体，为数据流提供智能化处理能力，具体配置与应用如下：

1. **大模型调用**  
   - **附件配置**（非必填，可添加多个附件，增强大模型理解能力）：  
     不同类型附件的配置规则存在差异，具体要求如下表所示：

| 附件类型 | 配置规则 |
| --- | --- |
| 图片类 | 默认自动选中，支持添加任意数量的图片（格式不限），大模型会自动识别图片内容并融入处理逻辑。 |
| 视频类 | 仅视觉模型支持添加视频，且**最多只能选择 1 个视频**；若已添加 1 个视频，其他附件的视频选择选项会灰化禁用，悬停时显示提示文案：“视觉模型只支持选择一个视频”，避免多视频处理冲突。 |
| 文件类 | 支持两种选择方式：1. 直接选择文件 / 文件变量：可指定文件版本（非必填，默认使用最新版本）；2. 输入 URL/URL 变量：无需指定文件版本，系统会通过 URL 自动获取文件内容（需确保 URL 可访问且无权限限制）。 |

   - **节点输出**：  
     大模型处理完成后，输出两种结果格式，满足不同场景需求：
     - answer：纯文本结果（例如：大模型生成的回答、分析结论等）；
     - json：从文本结果中自动解析出的 JSON 对象（若文本包含 JSON 格式内容，系统会自动提取；若无，则输出 null）。

2. **小模型 - Embedding**  
   - **核心任务**：将文本、图像、音频等非结构化信息，转换为数值形式的向量（一组连续数字），在数学空间中捕捉信息的语义特征，实现信息的可计算与可比较。
   - **典型应用场景**：
     - 检索：向量数据库搜索、语义搜索（例如：根据用户问题向量，匹配最相似的文档向量）、推荐系统（根据用户兴趣向量推荐相似内容）；
     - 聚类：将语义相似的文本自动分组（例如：将客户反馈按 “投诉”“建议”“咨询” 聚类）；
     - 分类：将向量作为特征输入至分类模型，实现文本分类（例如：垃圾邮件识别、情感分析）；
     - 去重：通过计算向量相似度，识别重复或高度相似的内容（例如：文档去重、图片去重）。
   - **配置与输出说明**：
     - 输入：需处理的内容数组（text array），至少包含 1 项内容（例如：["用户反馈内容1", "用户反馈内容2"]）；
     - 输出：data（Embedding 结果数组），与输入数组一一对应，每个输入内容对应一个向量数组（例如：输入 2 项内容，输出 2 个向量数组）。

3. **小模型 - Rerank**
   - **核心任务**：对初步筛选的候选结果（例如：通过 Embedding 检索得到的 Top-100 文档）进行精细化重排序，根据内容与查询的相关性，筛选出最匹配的少量结果，提升检索精度。
   - **典型应用场景**：
     - 检索增强生成（RAG）系统：在向量检索后，对 Top-K 结果重排序，将最相关的 3-5 个文档作为上下文提供给大模型，减少无关信息干扰，提升回答准确性；
     - 搜索引擎：对初步搜索结果按相关性重排序，优化用户搜索体验（例如：将用户最可能需要的结果排在前列）。
   - **配置与输出说明**：
     - **输入**：
       - query：查询文本（不能为空，例如：“如何配置数据流的定时触发？”）；
       - documents：候选文本数组（至少包含 1 项内容，例如：["定时触发配置步骤1...", "定时触发配置步骤2..."]）；
     - **输出**：
       - results：Rerank 详细结果（包含每个候选文本的相关性得分 relevance_score、排序后的索引等）；
       - documents：按 relevance_score 降序排列后的候选文本数组（相关性最高的文本排在首位）。


## 写入索引库
- **前置要求**：需提前在系统中创建目标索引库，创建路径为：【多模态数据湖】→【索引】→【索引库管理】，未创建索引库将无法完成节点配置。
- **配置步骤**：
  1. 在节点界面的索引库列表中，选择需写入数据的目标索引库（需具备该索引库的写入权限）；
  2. 配置索引内容：支持两种输入方式：
     - 引用数据源变量：选择前序节点输出的 JSON 格式变量（例如：“数据转换” 节点的 json_result 变量）；
     - 手动输入 JSON 字符串：直接编写符合索引库字段结构的 JSON（例如：{"id": "123", "content": "测试数据", "create_time": "2026-01-05"}）；
  3. 点击 “验证” 按钮，确认 JSON 格式与索引库字段匹配（若格式错误或字段不匹配，系统会提示具体问题，需修改后重新验证）。


## SQL 写入
- **配置步骤**：
  1. **源数据设置**：
     - 支持两种数据格式：array 数组（批量数据）、json 对象（单条数据）；
     - 数据来源：可手动输入数据（例如：[{"name": "张三", "age": 30}, {"name": "李四", "age": 28}]），或引用前序节点的输出变量（例如：“数据提取” 节点的 data_list 变量）。
  2. **批量写入大小**：设置单次 SQL 批量插入的数据量（例如：设置为 1000，表示每次向数据库插入 1000 条数据），需根据数据库性能合理调整，避免因单次插入数据量过大导致超时。
  3. **数据库连接配置**：
     - 连接类型：支持多种数据库类型，包括 mariadb、mysql、dameng（达梦）、postgresql、sqlserver、oracle、人大金仓，需选择与目标数据库匹配的类型；
     - 连接名称：从已配置的数据库连接列表中选择目标连接（需提前在系统 “数据连接” 模块创建连接并测试连通性，未配置的连接无法选择）。
  4. **目标表配置**：
     - 两种模式可选：
       - 新建目标表：自定义表名（需符合数据库表名规范，例如：user_info_2026），系统会根据源数据的字段结构自动生成表字段（若源数据为 json 类型，支持自动映射字段名与类型，无需手动输入）；
       - 选择已有目标表：从数据库的现有表中选择目标表，表名与字段映射不可修改（需确保源数据的字段与目标表字段匹配，否则会导致写入失败）。
  5. **表信息补充**：
     - 表名：若为新建表，需输入符合规范的表名；若为已有表，自动显示表名，不可修改；
     - 表概述（非必填）：输入表的用途描述（例如：“2026 年用户信息表”），便于后续管理与维护；
     - 字段映射：若源数据为 json 类型，系统会自动映射字段（例如：json 的 name 字段映射为表的 name 字段），也可手动调整映射关系（例如：将 json 的 user_name 字段映射为表的 name 字段）。


## 文档库写入
支持两种文档操作：“新建文件” 与 “更新文件”，不同操作支持的文件格式存在差异，具体如下表所示：

| 操作类型 | 支持文件格式 | 操作说明 |
| --- | --- | --- |
| 新建文件 | docx（Word 文档）、xlsx（Excel 表格）、pdf（PDF 文档）、markdown（MD 文档） | 在指定文档库文件夹中，创建新文件并写入内容（内容可引用变量或手动输入），新建文件会生成唯一文件 ID，便于后续查询与管理。 |
| 更新文件 | docx、xlsx、markdown | 对文档库中已存在的文件进行内容更新，需指定目标文件（可通过文件 ID 或文件路径选择），更新后会保留文件历史版本（支持回滚至历史版本），pdf 格式文件不支持更新操作。 |


## 内容处理
提供四类内容处理功能，覆盖文件文本提取、格式转换等核心场景，具体配置与说明如下：

| 操作类型 | 功能说明 | 配置与输出 |
| --- | --- | --- |
| 提取文件中的所有文本 | 从指定格式的文件中，批量提取文本内容（忽略格式信息，仅保留纯文本） | 配置：1. 选择目标文件（可引用文件变量或直接选择文档库中的文件）；2. 选择文件版本（选填，默认使用最新版本，若需提取历史版本文本，可手动选择版本号）；<br>支持格式：.docx、.dotx、.dot、.doc、.odt、.wps、.docm、.dotm、.dwg、.xlsx、.xlsm、.xlsb、.xls、.et、.xla、.xlam、.xltm |
