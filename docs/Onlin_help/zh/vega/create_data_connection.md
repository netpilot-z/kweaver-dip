# 如何创建数据连接？
本文档将指导您在 DIP 平台的VEGA 虚拟化模块中创建数据连接，包括数据连接类型介绍、通用创建流程、典型场景配置及后续扫描管理操作，帮助您实现多源数据的统一接入与管理。

> **前提条件**
1. 您已拥有 DIP 平台的访问权限，且具备VEGA 虚拟化模块的操作权限（如 数据连接管理、扫描管理权限）。
2. 待接入的数据连接（如 MySQL 数据库、AnyShare 存储）已正常部署并可访问，且已获取数据源的连接信息（如 IP 地址、端口、账号密码）。


## 数据连接介绍
VEGA 虚拟化模块支持**统一数据连接管理**，可接入结构化数据、非结构化数据及其他类型数据源，为多源数据整合提供基础支持。通过数据处理引擎层，可从不同系统渠道提取数据并智能化融合，确保决策依据的全面性与准确性。

### 支持的数据连接类型
| 数据连接分类 | 具体类型 | 连接方式（默认） |
| --- | --- | --- |
| 结构化数据 | MySQL、MariaDB、Oracle、PostgreSQL、SQLServer、Apache Doris、Hologres、openGauss、Dameng、GaussDB、MongoDB、Apache Hive、ClickHouse、TDH Inceptor、MaxCompute | JDBC（Hive 可选 Thrift） |
| 非结构化数据 | Excel、AnyShare 7.0 | HTTPS（Excel 依存储介质适配） |
| 其他 | 听云、OpenSearch | 平台适配协议 |


## 数据连接通用创建流程
无论哪种数据连接类型，均可按照以下 2 个步骤创建数据连接，后续典型场景将补充各类型的特殊配置说明。

### 进入数据连接管理页面
1. 登录 DIP 平台，进入「VEGA 虚拟化」>「数据连接」>「连接管理」页面；
2. 点击【新建】，进入数据连接新建页面。

### 选择数据连接类型并配置连接信息
1. 在 “选择连接类型” 步骤，根据实际需求从左侧分类列表中选择对应的数据连接类型（如 “结构化数据 > MySQL”），点击【下一步】。
2. 进入 “详细配置” 步骤，根据页面提示填写连接信息（不同数据连接类型的配置项略有差异，典型场景将详细说明），示例配置项包括：
    ○ 基础信息：数据连接名称（自定义，建议包含数据连接用途，如 “MySQL - 生产 - 订单库”）、描述（可选，记录数据连接用途或备注）。
    ○ 连接信息：IP 地址 / 域名、端口、数据库名称（或实例名）、账号、密码。
    ○ 高级配置（规划中）：超时时间、字符编码、SSL 配置（如需加密连接）等。
3. 点击【测试连接】按钮测试连接是否可用：
    1. 若提示 “连接失败”，请检查以下内容：
    ○ 数据连接的 IP 地址、端口是否正确，且网络是否通畅（可通过 ping 命令或 telnet 命令验证）。
    ○ 填写的账号密码是否正确，且该账号具备访问数据源的权限（如 MySQL 账号需有 “SELECT” 权限）。
    ○ 数据连接是否正常运行（如数据库服务是否启动、AnyShare 服务是否可用）。
    2. 若测试连接通过，则数据连接可用，点击【确定】完成数据连接的新建操作。

> **说明**
在「连接管理」页面，您还可对已创建的数据连接执行以下操作：
- 查看：点击【查看】，查看数据连接的详细配置与关联信息。
- 编辑：点击【编辑】，修改数据连接的配置信息（修改后需重新测试连接）。
- 删除：点击【删除】，（需确认无关联扫描任务或数据视图，否则无法删除）。
- 搜索：通过页面顶部的搜索框，按 “数据连接名称”“数据连接类型” 筛选目标连接。


## 数据连接创建典型场景
本节将针对常用数据连接类型，补充特殊配置说明与示例，通用步骤可参考 “数据连接通用创建流程”。

### 结构化数据 - MySQL 数据源创建
配置信息示例（以 “服务管理系统数据库” 为例）：

| 配置项 | 示例值 | 说明 |
| --- | --- | --- |
| 数据连接名称 | MySQL - 服务管理系统 | 自定义，便于识别 |
| 数据源类型 | 结构化数据 > MySQL | 固定选择 |
| 主机地址 | 192.168.1.100 | MySQL 服务器 IP 地址或域名 |
| 端口 | 3306 | MySQL 默认端口，若已修改需填写实际端口 |
| 数据库名称 | service_db | 需连接的 MySQL 数据库实例名 |
| 账号 | db_user | 具备该数据库访问权限的 MySQL 账号 |
| 密码 | ******** | 对应账号的密码 |
| 连接方式 | JDBC | 默认，无需修改 |

### 非结构化数据 - Excel 数据连接创建
Excel 数据连接需根据**存储介质**选择配置方式，支持 “AnyShare”（外部存储）与 “文档库”（DIP 内部存储）两种介质。

#### 存储介质选择 “AnyShare”（接入外部 AnyShare 数据）
| 配置项 | 示例值 | 说明 |
| --- | --- | --- |
| 数据连接名称 | Excel - 爱数技术资料 - AnyShare | 自定义 |
| 数据源类型 | 非结构化数据 > Excel | 固定选择 |
| 存储介质 | AnyShare | 固定选择 |
| 连接地址 | anyshare.aishu.cn | AnyShare 服务的 IP 地址或域名 |
| 端口 | 443 | AnyShare 默认 HTTPS 端口 |
| 用户 ID | app_tech | AnyShare 应用账号（需在 AnyShare 管理控制台创建） |
| 密码 | ******** | 应用账号密码 |
| 存储路径 | 技术文档 / Excel 资料 | 仅支持部门文档库或自定义文档库，无需 AnyShare 前缀；可指定单个.xlsx 文件或目录（目录仅接入一级子目录下的.xlsx 文件） |

#### 存储介质选择 “文档库”（接入 DIP 内部数据）
| 配置项 | 示例值 | 说明 |
| --- | --- | --- |
| 数据连接名称 | Excel - 技术资料 - 文档库 | 自定义 |
| 数据连接类型 | 非结构化数据 > Excel | 固定选择 |
| 存储介质 | 文档库 | 固定选择 |
| 存储路径 | 个人文档库 / 技术资料 | 仅支持个人文档库，无需 AnyShare 前缀；可指定单个.xlsx 文件或目录（目录仅接入一级子目录下的.xlsx 文件） |

#### 注意事项
1. 访问权限：需使用 “文档库对应的用户” 操作数据连接（如个人文档库需用本人账号），否则会提示 “文件无权限”。
2. AnyShare 版本：仅支持 AnyShare 7.0 及更早版本；Proton 3.0 及以上版本中，应用账号的创建与授权方式变更，暂不支持指定部门 / 自定义文档库，后续方案待定。
3. 文件类型：仅支持.xlsx 格式文件，不支持.xls 或其他格式。


## 查看数据连接详情
创建数据连接后，可通过以下步骤查看详细信息，便于验证配置或排查问题：
1. 进入「VEGA 虚拟化」> 「数据连接」> 「连接管理」页面；
2. 在数据连接列表中，找到目标连接，点击操作列的【查看】按钮；
3. 在详情页面中，可查看以下信息：
    ○ **基础信息**：数据连接名称、类型、创建人、创建时间、更新时间等。
    ○ **表属性**（结构化数据）：关联的数据库表名称、表类型、表描述等。
    ○ **字段属性**（结构化数据）：表的字段名、数据类型、是否主键、字段描述等。
    ○ **连接配置**：完整的连接信息（密码已脱敏）。


## 扫描管理（数据接入后续操作）
创建数据连接后，需通过 “扫描管理” 将数据源的表 / 文件接入平台，以便后续用于数据视图、数据模型的构建。支持 “使用整个数据连接创建” 与 “使用数据连接中的表创建” 两种扫描方式。

### 使用整个数据连接创建扫描任务（通用方式）
适用于需要接入数据源中所有表的场景，步骤如下：
1. 进入「VEGA 虚拟化」> 「数据连接」>「扫描管理」页面；
2. 点击页面右上角的【+ 新建扫描任务】，在下拉菜单中选择【使用整个数据连接创建】；
3. 在弹出的对话框中，从下拉列表选择需要扫描的数据源（即已创建的数据连接），点击【开始扫描】；
4. 扫描过程中，页面将展示实时进度（如 “已扫描 3/10 张表”），**注意：刷新或关闭页面会终止扫描**，已扫描的数据源会保留，未扫描的将停止；
5. 扫描终止或完成后，页面将弹窗提示 “扫描成功的表数量”（如 “成功扫描 10 张表”）；
6. 返回「扫描管理」页面，列表将展示所有创建的扫描任务，包括扫描状态（成功 / 失败）、扫描情况（扫描数量 / 总数量）、创建人、创建时间等信息。

### 使用数据连接中的表创建扫描任务（仅 OpenSearch 适用）
适用于仅需接入 OpenSearch 数据源中部分表的场景，步骤如下：
1. 进入「VEGA 虚拟化」 > 「扫描管理」页面。
2. 点击【+ 新建扫描任务】，选择【使用数据连接中的表创建】。
3. 在对话框中，先选择目标 OpenSearch 数据连接，再从下方的 “可选表列表” 中勾选需要扫描的表（支持多选），点击【开始扫描】。
4. 后续扫描进度查看、结果提示与上述一致。


## 常见问题
Q1：**测试数据连接时提示 “连接超时”，如何处理？**
A1：请按以下步骤排查：
- 验证数据源服务器是否正常运行（如 MySQL 服务是否启动、AnyShare 是否可访问）。
- 检查 DIP 平台服务器与数据源服务器之间的网络是否通畅（可通过 ping 数据源 IP、telnet 数据源端口验证，如 `telnet 192.168.1.100 3306`）。
- 确认数据源端口未被防火墙拦截（在数据源服务器防火墙中开放目标端口，允许 DIP 平台 IP 访问）。

Q2：**扫描任务终止后，已扫描的数据连接是否需要重新扫描？**
A2：请按以下步骤处理：
- 无需重新扫描全部数据，已扫描成功的数据源会保留在「扫描管理」列表。
- 若需补充扫描未完成部分，可重新创建扫描任务并选择相同数据连接，平台会自动跳过已扫描内容，仅扫描未完成部分。
