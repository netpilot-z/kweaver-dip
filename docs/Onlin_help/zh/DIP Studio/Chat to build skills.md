# 如何通过会话方式创建技能
通过创建技能，你可以让数字员工获得更明确、可复用的执行能力。技能可将外部API、数据库查询或复杂计算逻辑，封装为数字员工可理解、可调用的能力单元，助力其在业务场景中专业、稳定完成任务。

例如：可创建「查询客户订单」「调用外部接口生成报告」等技能。配置完成后，数字员工可按需调用，提升复杂任务处理效率与准确率。

创建技能后，数字员工可实现：
- **更专业**：围绕特定业务场景，精准理解与处理工作
- **可复用**：沉淀验证有效的流程为标准能力，反复调用
- **可组合**：多技能协同，完成多步骤、多环节复杂业务流程

本文将手把手指导：描述需求→生成技能→预览测试→正式发布全流程。

## 先决条件
开始创建技能前，需满足以下条件：
- 拥有 **KWeaver DIP 平台管理员账号密码**
- 可正常访问 **KWeaver DIP 平台登录页面**
- 技能如需调用外部接口，提前备好：
  - API文档
  - 访问凭证（API Key、Token等）
  - 输入参数&返回结果说明

## 进入技能配置页面
1. 进入「KWeaver DIP」→「我的数字员工」
2. 点击【新建】，进入数字员工新建页面
3. 在新建页面选择「技能配置」
4. 右侧区域点击【选择技能】，进入技能选择&创建界面

## 描述技能需求并生成技能
支持自然语言描述，系统自动理解并生成技能内容：
1. 在「选择技能」弹窗找到文本输入框
2. 填写技能需求，建议写清：
   - 技能目标
   - 执行流程
   - 所需输入信息
   - 预期输出结果
3. 可上传配套资料补充说明：
   - 业务说明文档
   - 示例数据
   - 接口资料
   - 规则说明文件

### 需求补充确认
若系统对技能边界理解不全，会自动抛出**3-5个确认问题**，聚焦：
- 技能适用范围
- 输入输出规范
- 核心处理规则
- 异常兜底方案
按需补充信息，保障生成结果精准。

### 自动生成技能文件
系统完全理解需求后，自动完成：
- 生成标准化技能文件包
- 撰写完整技能内容
- 输出可测试、可发布的技能初稿

## 预览和测试技能
技能生成后，务必先预览测试，校验是否符合业务预期：
1. 在技能配置页，通过对话触发测试
2. 在右侧预览区查看执行结果
3. 重点核验4项核心内容：
   - 是否正确识别输入信息
   - 执行流程是否贴合需求
   - 输出结果完整、准确
   - 无遗漏边界场景

> 💡提示：测试不符预期时，返回需求描述区，补充/优化说明后重新生成，提升技能质量。

## 发布技能
测试全量通过后，正式上线发布：
1. 点击页面右上角【发布】按钮
2. 发布前二次核对：
   - 技能名称
   - 技能描述
   - 版本信息
3. 确认无误，完成发布操作

## 重要提示
1. 技能描述越细化，自动生成效果越精准
2. 依赖外部接口的技能，务必提前打通权限、完善参数&返回结构说明
3. 上线前必须完成至少1次全流程测试，规避执行偏差
