# smart-search-tables（找数子技能）

用于定位数据资产：表、视图、字段、归属职责与候选数据集。

## 参考经验

- [`smart-search-tables-knowledge.md`](smart-search-tables-knowledge.md)：沉淀找表阶段常见问题与防漏策略（如结果截断导致漏表、精确表名二次校验）。
- [`smart-search-tables-playbook.md`](smart-search-tables-playbook.md)：通用防漏检索模板与最小执行示例（按需加载）。
- [`smart-search-tables-step7-commands.md`](smart-search-tables-step7-commands.md)：步骤 7 命令模板与参数防错细则（按需加载）。

## 使用场景

- 用户问“这类数据在哪张表/视图”
- 需要确认字段归属、治理职责、元数据位置
- 问数前置：先找表再收敛查询口径

## 输入要求

- `kn_id_find_table`（必须来自 `SOUL.md`）
- 业务主题关键词（如企业、处罚、联络员）
- 可选过滤：系统域、部门、时间范围

## 强约束（SOUL 对齐，必须执行）

- 必须严格按第 5-10 步顺序执行，不得跳步、并步、倒序或绕过门禁。
- 禁止编造或篡改流程：不得虚构已执行步骤、不得伪造步骤结果、不得擅自修改流程定义与执行记录。
- 任一步骤失败必须立即停止流程并返回真实失败原因；在失败状态下不得继续后续步骤。

## 编排门禁流程（承接总入口第 4 步后继续，序号连续）

进度执行硬约束（必须执行）：
- 每完成第 5-10 步中的任一步，都要立即输出一次进度。
- 进度模板固定为：`[smart-search-tables] 进度：已完成第 N 步（步骤名称）；下一步：第 N+1 步（步骤名称）`
- 若当前步骤尚未输出进度，**不得进入下一步**。
- 若发现缺步、跳步或步骤失败，必须**立即停止流程**并说明原因，不得继续执行。
- 若流程在第 10 步结束，必须输出：`[smart-search-tables] 进度：已完成第 10 步（总结结果）；流程完成`

5. **确认知识网络**：使用 `kweaver bkn get <kn_id_find_table>` 确认知识网络存在；并检查该网络为**元数据知识网络**。若不存在或不是元数据知识网络，先提示并要求切换后再执行。
6. **配置知识网络**：使用 `kweaver context-loader config set --kn-id <kn_id_find_table>` 配置当前检索知识网络。
7. **查询数据（样例命令）**：按当前终端环境选择对应模板执行实例检索；将模板中的“用户问题”替换为用户输入的找数问题。  
   步骤 7 的完整命令模板与参数防错细则见：[`smart-search-tables-step7-commands.md`](smart-search-tables-step7-commands.md)（按需加载）。

   **限制规则（必须遵守）**：
   - 第 7 步必须优先执行 `kweaver context-loader query-object-instance`。
   - 若命令执行失败（如 `Invalid JSON argument`、权限/网络异常等），只返回原始报错并停止当前流程。
   - 未经用户明确同意，不得自动切换为 `kn-search` 或其它检索命令兜底。
8. **筛选视图**：根据 `description`、视图业务名称、字段名称筛选视图。
   - **防漏硬约束（必须遵守）**：
     - 若命令输出出现 `output truncated`、结果体明显过长、或存在“可能遗漏”的用户反馈，必须执行“二次校验”后才能进入第 9/10 步。
     - 二次校验最少包含以下动作：
       1) 使用“用户给出的精确表名/短语”再次检索（`limit` 建议 30）；
       2) 若仍疑似不全，按主题分词（如“科研服务/展览/教育活动/社区服务”）分批检索并去重合并；
       3) 在最终结果中显式说明“已执行防截断二次校验”。
     - 未执行上述二次校验，不得宣称“完整清单”。
9. **查询职责信息（按用户问题触发，必要时必须执行）**：当用户问题包含或指向“归属/职责/负责/管理/监管/审批/信息系统/部门”等治理归属意图，或显式提到“由某部门/某单位管理范围”等表述时，获取部门、信息系统、职责信息。使用以下命令（不触发时可跳过）：

   ```bash
   kweaver context-loader config set --kn-id duty
   kweaver context-loader kn-search "职责"
   ```

10. **总结结果**：总结检索结果，列表以表格方式展示；最终结果中必须明确展示每个候选对象的**视图业务名称**与**视图技术名称（表名）**。

## 输出要求

1. 候选表/视图清单（必须包含：视图业务名称、视图技术名称/表名）
2. 关键字段与推荐优先级
3. 归属或职责信息（如可得；仅当本流程已执行步骤9）
4. 下一步问数建议（可选）
5. 最终结果中**不得**输出任何 SQL 语句、SQL 模板，且**不得**建议用户“使用某条 SQL 查询”；若用户需要取数，统一引导到 `smart-ask-data` 分支。

## 不做事项

- 不直接产出最终指标结果
- 不替代问数输出 SQL + 数据
- 不在找表结果中展示 SQL，或给出“建议使用某条 SQL 查询”的内容
- 不在无 KN 或无命中时编造候选

## 失败处理

- 返回无命中或权限问题的真实原因
- 引导用户补充关键词/业务范围
- 不切换到问数分支伪造结果
