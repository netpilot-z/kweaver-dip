# 能力缺口检查清单

## 1. 需求拆解

- 新能力，还是已有能力的新阶段？
- 主路径能力，还是失败回退能力？
- 编排能力，还是执行能力？
- 一次性任务，还是会重复出现的工作流？

## 2. 覆盖比对

- 已有 skill 是否已完整覆盖？
- 是否只缺少少量规则或 references？
- 是否已存在相似 skill，只是 description 不够好？

## 3. 生成判断

只有满足任一才建议新建：
- 存在明确能力缺口，且会重复复用
- 现有 references 已形成独立触发价值
- 需把稳定流程沉淀为可触发能力

## 4. 不应生成

- 只是修改现有 skill 的 wording
- 只是补一两条规则
- 只是一次性需求
- 只是外部 skill 的直接能力调用

## 5. 拆分原则

- 主路径 vs 失败收敛分开
- 编排 vs 执行分开
- 外部能力包装层 vs 内部能力层分开

## 6. 外部边界

以下能力不改写：create-bkn / kweaver-core / data-semantic
围绕这些只能做：编排层、桥接层、包装层。
