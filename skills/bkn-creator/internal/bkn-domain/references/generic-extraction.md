# 通用提取规则

领域未识别或领域知识不足时的通用提取流程。

## 步骤

1. 切分输入为"对象候选语句"和"关系候选语句"
2. 从名词短语提取对象类；过滤纯属性词（状态、名称、备注）
3. 从动词/动宾短语提取关系类（"提交申请""关联订单""依赖库存"）
4. 为每个对象推断候选主键（优先编码/单号/ID）
5. 为每个关系补齐方向、基数和证据

## 对象判定

- 可独立存在 + 可被唯一标识 + 生命周期超一次动作 → 对象
- 临时动作/单次结果/纯展示字段 → 不是对象
- 名称含 `→`/`:`/`/`/`，`/`与` → 排除候选

## 关系方向判定

1. 溯源视角优先：目标对象有来源字段（src*/source*/parent*/root*）→ 当前 → 来源
2. 驱动视角次之：上游驱动下游 → 驱动 → 被驱动
3. 两种均合理且冲突 → 标记"方向歧义"，请用户确认

## 质量检查

- 对象名去重（同义词合并为主名 + aliases）
- 关系名去重（语义一致合并）
- 每条关系引用的对象必须存在于对象总清单
- 推断对象必须含 推断理由
- 无法确认主键/基数时标记 unknown，不臆造
