# 显示规范

## 回显模板

```
### <标题>（<类型>）
说明：
- <要点>
下一步：<动作或确认短语>
```

标题用自然中文。确认类必须含可执行选项。

## 展示策略

- 默认仅输出摘要（数量、状态、风险、下一步）
- 同一轮只用一种主格式（文本摘要 / 表格 / YAML 三选一）
- 详情仅在用户明确请求时输出
- skill 内部产物静默计算，仅最终摘要对外

### 快速模式

**适用场景**：输入为结构化文档（路径 A/A-fast），建模清单清晰、无 pending 对象、无冲突关系。

**快速模式下 pipeline 的展示规则**：

| 阶段 | 展示内容 |
|------|---------|
| 阶段一（建模） | 仅展示最终提取摘要（对象数、关系数、规则数），不展示完整 YAML 清单；直接请用户确认 |
| 阶段二（落盘） | 仅展示落盘摘要（文件数、对象/关系/分组数量），validate 通过后直接请用户确认 |
| 阶段三（环境） | 仅展示 bind_mode 判定结果 |
| 阶段四（绑定） | 仅展示绑定决议摘要（bound/pending/rejected 数量），不展示逐字段比对 |
| 阶段七（推送） | 仅展示门禁结果和推送状态 |

**默认行为**：由 pipeline 在阶段一策略确认后判定是否启用快速模式（路径 A/A-fast 且无 pending/冲突时启用）；用户可随时说"看详情"/"展开"查看完整中间产物。

**非快速模式**（路径 B/C 或存在 pending/冲突/高风险规则）：保持原有展示策略，在关键决策点展示详细中间产物供用户审核。

## 字段显示词典

| 内部编码 | 中文 | | 内部编码 | 中文 |
|----------|------|-|----------|------|
| `keep` | 保留 | | `mapped` | 已映射 |
| `add` | 新增 | | `waived` | 已豁免 |
| `rename` | 重命名 | | `blocked` | 待处理 |
| `merge` | 合并 | | `core_business` | 核心业务 |
| `drop_candidate` | 建议移除 | | `view_derived` | 视图衍生 |

## 约束

- 禁止 emoji
- ID 不截断
- 百分比保留 1 位小数
- Description 仅写稳定业务语义，映射不确定信息放独立字段
