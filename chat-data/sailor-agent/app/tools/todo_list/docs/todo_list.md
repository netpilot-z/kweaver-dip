# 任务拆分工具
这是一个将用户问题拆成一个个不可分割的任务列表的工具

## 参数
用户问题：由意图理解工具丰富后的用户问题
问题场景：由意图理解工具得出的用户问题场景
拆解策略：指定当前场景下的拆解策略
会话ID：当前会话的ID，用来获取当前会话的任务列表


## 任务拆解步骤
1：通过会话ID获取历史任务列表（如果为空或已完成，则视为无历史任务）
2：结合当前用户问题、历史任务和拆解策略，让大模型生成一份完整的任务列表
3：将任务列表标记为未开始状态，同时保存问题和任务列表到 Redis，然后返回


###  获取会话任务列表逻辑
1. 通过会话ID查询redis得到任务列表
2. 如果查不到或者任务已经完成，返回空列表
3. 如果查到是未完成状态，将任务列表返回


### 大模型任务拆解
#### 入参
1. 历史任务列表，可能为空
2. 当前用户问题
3. 场景与拆解策略

#### 新生成逻辑
1. 让大模型根据当前问题、历史任务（可为空）和拆解策略，生成一份完整的任务列表
2. 所有任务的 `status` 初始为 `pending`，`blockedBy` 只允许引用已存在的任务 `id`


#### 调整任务列表
1. 根据输入的会话id查询历史任务列表
2. 参数将拆解策略里是调整原因
3. 根据输入的调整原因让大模型调整任务列表，注意已经执行完的任务不变
4. 将重新生成的任务列表保存到redis

### 缓存策略
1. string结构，24小时过期
