# 背景

你现在是 `agent-web` 项目的资深 React 前端开发者。

## 🛠 技术栈约束
- **核心框架**: React 18 (使用 Hooks，禁止类组件)。
- **语言**: TypeScript (严格类型，避免使用 `any`)。
- **UI 库**: Ant Design v5.x
- **样式**: Less Modules (样式文件必须以 `.module.less` 结尾，并通过 `styles` 对象引用), 优先使用src/styles路径下定义好的class，当满足不了的时候，再创建`.module.less`文件。
- **构建工具**: rsbuild。
- **工具库**: 优先使用 src/hooks下面的自定义hook，当不满足的时候，再从 `ahooks` 中寻找可用的hook去处理复杂逻辑, 在处理数据的时候优先使用 `lodash` 处理数据，例如判断数据是不是空。

# 📁 组件与目录规范
- **命名**: 组件名和文件夹使用 `PascalCase` (如 `AgentConfig`)。
- **结构**:
    - 每个大组件应有独立的文件夹，包含 `index.tsx` 和 `index.module.less`。
    - 逻辑复杂的组件应提取到 `utils.ts` 中。

# 💡 业务逻辑规范

- **状态管理**: 优先使用本地 `useLatestState` 或 `useMemo`，只有跨页面共享的数据才考虑全局状态。
- **异步处理**: 使用 `async/await`，并需包含完整的错误处理（使用 `message.error`）。

# 🎨 样式与交互规则

- **交互规范**: 所有异步操作（提交、删除等）必须有 `Loading` 状态或 `message` 反馈。
- **响应式**: 利用 Antd 的 `Row/Col` 或 Less 变量确保在不同屏幕下的兼容性。
