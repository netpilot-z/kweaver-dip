# KWeaver DIP 0.5.0 版本正式发布
KWeaver DIP 0.5.0 版本以使用体验全面提升为核心，重点围绕部署灵活性、业务知识网络状态主动监测、数据分析员找数问数准确性提升进行优化。

## 核心亮点速览
- **DIP Studio**：新增 OpenClaw双部署模式与技能包元数据校验，优化导航结构与页面跳转体验，有效降低平台部署与接入门槛。
- **BKN Creator**：支持从 PRD 自动提取业务规则并生成Skill，引入定时巡检与状态检测机制，实现业务知识网络状态主动监控。
- **数据分析员**：全面优化找数问数的语义识别与结果准确性，新增多义字段澄清、异常循环中断及 Token 自动续期等机制，保障查询稳定可靠。

## DIP Studio
本版本围绕部署灵活性、技能上传校验与操作体验三个方向持续迭代，实现以更低门槛完成平台接入，提升用户使用体验。

1. **OpenClaw 双部署模式支持**
支持以两种方式接入 OpenClaw，用户可在自有主机上独立部署，也可直接使用 DIP Studio镜像自带容器化OpenClaw，两种模式可按实际基础设施需求灵活选择，降低部署门槛。

2. **上传技能包元数据校验**
支持在上传技能包时自动检测 SKILL.md 文件的头部元数据完整性。若元数据缺失，系统将拒绝上传并提示用户补充必要信息，避免技能上传成功但无法被 OpenClaw 识别的问题，减少用户排查成本。

3. **全局业务知识网络与系统控制台页面跳转免登录**
支持用户从DIP Studio直接跳转至全局业务知识网络与系统工作台页面，无需重复进行登录认证，提升用户日常操作流畅度。

4. **领域知识网络卡片化展示**
已创建的领域知识网络支持以卡片形式在页面集中展示，用户可直观查看各领域知识网络的基本信息、修改者与更新时间。

5. **导航结构调整与优化**
优化全局业务知识网络页面导航，全新导航下包含：领域知识网络、通用知识网络、决策智能体、执行工厂和Autoflow模块；其中决策智能体模块下调整为"开发"与"广场"两个子页面，原API及模板页面统一迁移至广场页面。

6. **Bug 修复**
- 修复本地部署 OpenClaw 3.11后，创建平台预置BKN数字员工时报错的问题。
- 修复在平台中发布新数字员工时，预览计划文档出现报错的问题。
- 修复dip-hub界面初始化时修改 OpenClaw 配置不生效的问题，同步优化部署脚本与界面初始化重复填写 OpenClaw 信息问题，减少重复录入。
- 修复数字员工历史记录列表中标题展示异常的问题。
- 修复对话中上传文件后，历史记录页面未展示对应文件的问题。
- 修复数字员工执行过程中，工具输入与输出内容未在界面展示的问题，支持查看完整工具调用详情。
- 修复配置数字员工时，无法新建或选择业务知识网络的问题。
- 修复计划执行成功后，执行成果展示异常的问题，确保状态与实际执行结果一致。
- 修复数字员工对话页面复制内容时出现报错的问题，支持一键复制对话内容。

## BKN Creator
本版本新增基于PRD提取的业务规则生成Skill机制，并引入定时任务与业务知识网络检测能力，实现业务知识网络的动态问题发现与改进建议。

> 注：以下功能为企业版能力。

1. **基于PRD提取的业务规则生成Skill**
支持在创建领域知识网络时，自动从需求文档（PRD）中提取业务规则，并将其封装为可复用的Skill；用户提问时，系统通过Skill在业务知识网络上进行定向查询，使业务规则真正参与推理与响应过程，有效降低因规则缺失导致的幻觉风险。

2. **创建业务知识网络巡检任务**
支持为业务知识网络配置定时任务，按设定周期自动触发指定操作，实现业务知识网络的周期性运维监控。用户无需手动介入，平台即可在预定时间完成任务调度与执行。

3. **业务知识网络状态检测**
支持对业务知识网络发起定期巡检，自动验证数据映射准确性、数据连接状态及业务知识网络整体可用性，将业务知识网络的状态管理从"被动发现问题"转变为"主动预防风险"，确保数字员工在执行任务时始终依赖准确、可用的知识底座。

## 数据分析员
1. **找数问数结果准确性优化**
优化找数结果与业务知识网络实际数据的匹配逻辑，解决返回不存在数据表、应命中的相关表未出现在找数结果中，以及问数返回重复数据结果及模型幻觉的问题，确保找数问数结果准确、可复核。

2. **找数表名与数据完整性优化**
优化找数结果的展示与数据回填逻辑，解决表名被截断显示及返回UUID与实际数据不符的问题，确保找数结果信息完整、真实可溯。

3. **Token失效自动续期**
支持找数过程中检测到Token失效时自动重新获取并继续执行查询，无需用户手动干预，保障找数流程的稳定性与连续性。

4. **输出完整性优化**
解决找数与问数过程中回答突然中断、内容无法完整输出的问题，确保每次查询均能完整呈现分析结论与数据结果。

5. **特有名词语义识别与多义字段主动澄清**
支持识别提问中包含的行业特有名词简写，并正确匹配对应业务知识网络；当问数字段存在于多张表时，支持主动向用户发起澄清确认，而非直接选表作答，提升语义识别准确率与多义场景下的交互规范性。

6. **问数异常循环中断机制**
支持在问数过程中无法定位目标表时自动触发中断，避免因持续重复执行找表操作而陷入无效循环。

7. **问数跨表字段精准定位**
支持在所需字段存在于多张表的场景下，准确识别并选择包含目标字段的数据表进行查询，解决因错误选表导致字段找不到的问题，提升多表场景下问数的准确性与稳定性。
